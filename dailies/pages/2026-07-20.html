Paper 1

LongStraw: Long-Context RL Beyond 2M Tokens under a Fixed GPU Budget

Published: 2026-07-15

Link: http://arxiv.org/pdf/2607.14952

1. 📘 Topic and Domain: This paper is in the domain of large language model (LLM) systems and machine learning infrastructure, specifically focusing on long-context reinforcement learning (RL) post-training under fixed GPU memory budgets.
2. 💡 Previous Research and New Ideas: The paper builds upon Group Relative Policy Optimization (GRPO), Ring Attention, DeepSpeed-Ulysses, PagedAttention, FlashAttention, LoRA/QLoRA, and activation checkpointing. The new idea is an architecture-aware execution stack that separates prompt computation from response computation through a "capture once, replay the suffix" approach, retaining only model-specific conditional state to dramatically reduce live GPU memory during backward passes.
3. ❓ Problem: The paper addresses the growing gap between inference context lengths (approaching million tokens) and RL post-training context lengths (typically ≤256K), where GPU memory becomes a major bottleneck due to quadratic attention computation and long-lived backward states—particularly challenging for AI agents with accumulated long trajectories.
4. 🛠️ Methods: LongStraw evaluates the shared prompt once without automatic differentiation, retains only architecture-specific conditional state (GDN recurrent state and KV pages for Qwen; MLA latent and DSA index-key pages for GLM), and replays short response branches one at a time under autograd. It is implemented for two model families: Qwen3.6-27B (hybrid recurrent/full-attention) on 8 H20 GPUs, and GLM-5.2 (compressed attention MoE) on 32 H20 GPUs.
5. 📊 Results and Evaluation: On 8 H20 GPUs, LongStraw completes GRPO scoring and response backward for Qwen at 2.1M positions with group sizes 2 and 8, where increasing group size from 2 to 8 adds only 0.21 GB peak memory. A stress test extends Qwen to 4.46M positions. On 32 H20 GPUs, it validates end-to-end execution for a 2.1M-token prompt across all 78 layers of GLM-5.2. The results demonstrate that state lifetime and physical ownership—not sparsity alone—are key determinants of the practical context limit for RL post-training.
1. 📘 主题与领域: 本文属于大语言模型（LLM）系统与机器学习基础设施领域，专注于在固定GPU显存预算下进行长上下文强化学习（RL）后训练的研究。
2. 💡 先前研究与新思路: 本文基于组相对策略优化（GRPO）、Ring Attention、DeepSpeed-Ulysses、PagedAttention、FlashAttention、LoRA/QLoRA以及激活检查点等技术。新思路在于提出一种架构感知的执行栈，通过"一次性捕获、回复后缀"的方法将提示词计算与回复计算分离，仅保留模型特有的条件状态，从而在反向传播过程中显著降低GPU显存占用。
3. ❓ 问题: 本文解决了推理上下文长度（接近百万token）与RL后训练上下文长度（通常≤256K）之间日益扩大的差距，其中由于二次注意力计算和长生命周期的反向状态，GPU显存成为扩展训练上下文的主要瓶颈——这一挑战对需要累积长轨迹的AI智能体尤为突出。
4. 🛠️ 方法: LongStraw在禁用自动微分的情况下对共享提示词进行一次评估，仅保留架构特定的条件状态（Qwen的GDN循环状态和KV页面；GLM的MLA潜在量和DSA索引键页面），然后在自动微分下逐条重放短回复分支。该方法针对两类模型族实现：在8块H20 GPU上的Qwen3.6-27B（混合循环/全注意力），以及在32块H20 GPU上的GLM-5.2（压缩注意力MoE）。
5. 📊 结果与评估: 在8块H20 GPU上，LongStraw在210万位置、组大小为2和8的情况下完成了Qwen的GRPO评分和回复反向传播，组大小从2增加到8仅增加0.21 GB峰值显存。压力测试将Qwen扩展至446万位置。在32块H20 GPU上，对GLM-5.2的210万token提示词在全部78层上验证了端到端执行。结果表明，决定RL后训练实际上下文限制的关键因素是状态生命周期和物理所有权，而非单纯的稀疏性。

Function-Aware Fill-in-the-Middle as Mid-Training for Coding Agent Foundation Models

Published: 2026-07-13

Link: http://arxiv.org/pdf/2607.12463

1. 📘 Topic and Domain: This paper focuses on developing a mid-training methodology for coding agent foundation models within the field of large language models and automated software engineering.
2. 💡 Previous Research and New Ideas: Building on prior work in fill-in-the-middle (FIM) training, random-span FIM in code LLMs (Qwen-Coder, DeepSeek-Coder, StarCoder), structure-aware masking (AST-FIM), and agentic post-training pipelines (R2E-Gym, SWE-Smith, SWE-Lego), the paper introduces the novel insight that function call sites and coding agent steps share a structurally isomorphic four-stage decomposition (context, call/action, return/observation, continuation).
3. ❓ Problem: The paper addresses the training-time gap between left-to-right code pretraining and agentic post-training, where base models are rarely optimized for the bidirectional conditioning structure required to integrate external tool returns into ongoing agent reasoning.
4. 🛠️ Methods: The paper proposes function-aware FIM mid-training, which uses program dependency graph analysis with a complexity–inferability double criterion to select masking targets at function granularity, embeds chain-of-thought rationales generated by Gemini-3-Flash inside the FIM middle span, and trains on a 2.6B-token decontaminated corpus from 968 Python repositories before applying existing agentic post-training pipelines.
5. 📊 Results and Evaluation: The method improves SWE-Bench-Verified by +2.8/+3.0 points on Qwen2.5-Coder-Instruct 7B/14B and +3.2 on Qwen3-8B, with SWE-Bench-Lite gains of +3.7/+4.0/+5.4, generalizes across R2E-Gym and SWE-Smith pipelines, and additionally mitigates capability erosion on non-agent coding (LiveCodeBench +11.1) and tool-use benchmarks (τ-bench +3.9, BFCL +2.4), with the largest gains (+9.1 pp) concentrated on multi-function single-file reasoning tasks.
1. 📘 主题与领域: 本文聚焦于大语言模型与自动化软件工程领域，研究面向编码智能体基础模型的中间训练方法。
2. 💡 先前研究与新思路: 本文基于填空训练（FIM）、代码大模型中的随机跨度 FIM（Qwen-Coder、DeepSeek-Coder、StarCoder）、结构感知掩码（AST-FIM）以及智能体后训练流水线（R2E-Gym、SWE-Smith、SWE-Lego）等已有研究，提出函数调用点与编码智能体步骤共享同构的四阶段分解（上下文、调用/动作、返回/观察、延续）这一新见解。
3. ❓ 问题: 论文解决了从左到右的代码预训练与智能体后训练之间存在的训练鸿沟，即基础模型未针对将外部工具返回整合到智能体持续推理中所需的双向条件结构进行优化。
4. 🛠️ 方法: 论文提出了函数感知 FIM 中间训练，通过程序依赖图分析结合复杂度-可推断性双重准则在函数粒度上选择掩码目标，将 Gemini-3-Flash 生成的思维链推理嵌入 FIM 中间段，并基于来自 968 个 Python 代码仓库的 26 亿 token 去污染语料进行训练，再应用现有的智能体后训练流水线。
5. 📊 结果与评估: 该方法在 Qwen2.5-Coder-Instruct 7B/14B 上将 SWE-Bench-Verified 分别提升 +2.8/+3.0 分，在 Qwen3-8B 上提升 +3.2 分，SWE-Bench-Lite 提升 +3.7/+4.0/+5.4 分，跨 R2E-Gym 和 SWE-Smith 流水线均有效，并缓解了非智能体编码（LiveCodeBench +11.1）及工具使用基准（τ-bench +3.9、BFCL +2.4）的能力退化，最大增益（+9.1 个百分点）集中在多函数单文件推理任务上。
