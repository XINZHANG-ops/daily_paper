Paper 1

Harness Handbook: Making Evolving Agent Harnesses Readable,Navigable, and Editable

Published: 2026-07-13

Link: http://arxiv.org/pdf/2607.13285

1. 📘 Topic and Domain: This paper addresses software engineering for AI agent systems, specifically focusing on the evolution and maintenance of large-scale agent harnesses that coordinate foundation models, tools, and execution environments.
2. 💡 Previous Research and New Ideas: The paper builds on prior work in agent harness formalization (e.g., "Code as Agent Harness"), harness engineering/optimization, and repository representations for coding agents, and introduces a new idea of shifting from implementation-centric to behavior-centric representations that explicitly link system behaviors to their distributed source code.
3. ❓ Problem: The paper addresses the behavior localization problem—identifying all relevant implementation sites that must be modified in a large, tightly coupled harness when given a natural-language modification request, which is a central bottleneck for both human developers and coding agents.
4. 🛠️ Methods: The paper introduces the Harness Handbook, a three-level (L1–L3) behavior-centric representation built via static program analysis and LLM-assisted behavioral structuring, together with a Behavior-Guided Progressive Disclosure (BGPD) workflow that localizes edits from coarse handbook navigation to fine-grained source verification, plus automatic resynchronization after each code change.
5. 📊 Results and Evaluation: Evaluated on two open-source harnesses (Terminus-2 and Codex) with 30 modification requests each, Handbook-Assisted planning raised overall win rates by 10.0 and 18.9 percentage points, reduced planner token usage by 12.7% and 8.6%, and improved file/symbol-level F1 scores by 5.0–18.8 points against independent reference plans from stronger models.
1. 📘 主题与领域: 本论文聚焦于 AI 智能体系统中的软件工程问题，具体研究协调基础模型、工具和执行环境的大规模智能体框架（agent harness）的演化与维护。
2. 💡 先前研究与新思路: 本文基于智能体框架形式化（如 "Code as Agent Harness"）、框架工程与优化、以及面向编码智能体的代码仓库表示等已有研究，提出了一种从"以实现为中心"转向"以行为为中心"的新思路，即显式地将系统行为与其分布式的源代码实现进行关联。
3. ❓ 问题: 论文解决了行为定位（behavior localization）问题——在大型紧耦合的智能体框架中，给定一条自然语言的修改需求时，如何识别出所有需要修改的相关实现位置，这是人类开发者和编码智能体共同面临的核心瓶颈。
4. 🛠️ 方法: 论文提出了 Harness Handbook——一种通过静态程序分析与 LLM 辅助行为结构化构建的三层级（L1–L3）行为中心表示，以及行为引导的渐进式披露（BGPD）工作流，从粗粒度手册导航逐步细化到源代码验证，并在每次代码变更后自动重新同步。
5. 📊 结果与评估: 在两个开源智能体框架（Terminus-2 和 Codex）上各 30 条修改请求上评估，使用 Handbook 辅助规划将总体胜率分别提升 10.0 和 18.9 个百分点，规划器 token 消耗降低 12.7% 和 8.6%，并在文件和符号级上将 F1 分数相对更强模型的参考计划提升了 5.0–18.8 个百分点。

Paper 2

OvisOCR2 Technical Report

Published: 2026-07-14

Link: http://arxiv.org/pdf/2607.13639

1. 📘 Topic and Domain: The paper presents OvisOCR2, a compact 0.8B end-to-end model for page-level document parsing, situated in the fields of computer vision, document AI, and OCR.
2. 💡 Previous Research and New Ideas: Built upon Qwen3.5-0.8B and insights from prior work showing pipeline methods dominating document parsing benchmarks; the new idea is demonstrating that a single compact end-to-end model can surpass pipeline methods on parsing leaderboards through a combined data engine and multi-stage training recipe.
3. ❓ Problem: The paper addresses the performance gap where existing end-to-end document parsers lag behind pipeline methods, and tackles challenges of accurate text, formula, table, and reading-order recovery in a single deployable model with small parameter footprint.
4. 🛠️ Methods: The method combines a data engine with real-world filtered annotations and source-aligned synthetic HTML-rendered pages, followed by supervised fine-tuning, GRPO-based reinforcement learning with multi-component rewards (text, formula CDM, table TEDS) on a 4B branch, on-policy distillation transferring the teacher's behavior to the 0.8B student using student top-k reverse KL, and model fusion via weighted parameter averaging.
5. 📊 Results and Evaluation: OvisOCR2 achieves state-of-the-art overall score of 96.58 on OmniDocBench v1.6 (first end-to-end model to top this benchmark), 75.06 Avg3 on PureDocBench, and the highest overall score on an in-house benchmark across all difficulty tiers, with particularly strong performance on handwriting and complex-table subsets.
1. 📘 主题与领域: 论文介绍了OvisOCR2，一个用于页面级文档解析的紧凑型0.8B端到端模型，属于计算机视觉、文档AI和OCR领域。
2. 💡 先前研究与新思路: 基于Qwen3.5-0.8B以及先前关于流水线方法主导文档解析基准的研究；新思路在于通过结合数据引擎和多阶段训练方案，证明单个紧凑的端到端模型可以在解析排行榜上超越流水线方法。
3. ❓ 问题: 论文解决了现有端到端文档解析器性能落后于流水线方法的问题，并应对在小型可部署模型中准确恢复文本、公式、表格和阅读顺序的挑战。
4. 🛠️ 方法: 该方法结合了包含真实世界过滤标注和源对齐合成HTML渲染页面的数据引擎，随后进行监督微调、基于GRPO的多组件奖励（文本、公式CDM、表格TEDS）的4B分支强化学习、使用学生top-k反向KL将教师行为迁移到0.8B学生的在线策略蒸馏，以及通过加权参数平均进行的模型融合。
5. 📊 结果与评估: OvisOCR2在OmniDocBench v1.6上取得了96.58的最先进总体分数（首个登顶该基准的端到端模型），在PureDocBench上取得了75.06的最高Avg3分数，并在自建基准的所有难度层级上取得了最高总体分数，在手写体和复杂表格子集上表现尤为突出。

Paper 3

KnowAct-GUIClaw: Know Deeply, Act Perfectly, Personal GUI Assistant with Self-Evolving Memory and Skill

Published: 2026-07-14

Link: http://arxiv.org/pdf/2607.12625

1. 📘 Topic and Domain: This paper focuses on cross-platform GUI agent frameworks and personal AI assistants, specifically addressing GUI automation across mobile and desktop systems (Android, iOS, HarmonyOS, Windows) using LLM-based multi-agent architectures.
2. 💡 Previous Research and New Ideas: The work builds on OpenClaw-style agent frameworks, ReAct-style reasoning, dedicated GUI agents (CogAgent, UI-TARS, Aguvis), and memory/skill reuse systems (Reflexion, Voyager, ReasoningBank). Its new idea is the "Know Deeply, Act Perfectly" paradigm, instantiated as a four-stage Know–Route–Act–Reflect closed loop that decouples high-level orchestration from low-level GUI control with self-evolving memory and skills.
3. ❓ Problem: It addresses two core bottlenecks of mainstream OpenClaw-style personal assistants: (1) insufficient cross-platform GUI interaction capability, and (2) the absence of a built-in self-evolution mechanism that prevents continuous learning from execution experience.
4. 🛠️ Methods: The paper proposes the KnowAct-GUIClaw framework, featuring a host-executor two-tier collaboration, POMDP-formulated GUI control, attribution-aware persistent experience memory, a self-evolving state-validated skill library, a typed blackboard for cross-app information transfer, and trajectory distillation/reflection for skill and memory evolution.
5. 📊 Results and Evaluation: KnowAct-GUIClaw with Kimi-K2.6 achieves 64.1% success rate on MobileWorld GUI-Only (state-of-the-art, beating GPT-5.5 and Seed-2.0-Pro), 78.61% on AndroidDaily (resolved), and demonstrates transferability (8.5% gain with Kimi, 16.2% with Qwen3.5-35B), with cross-platform usability at 76.2% on HarmonyOS and 70.0% on Windows.
1. 📘 主题与领域: 该论文聚焦于跨平台 GUI 智能体框架和个人 AI 助手领域，具体研究基于 LLM 的多智能体架构在移动端和桌面端（Android、iOS、HarmonyOS、Windows）跨平台 GUI 自动化中的应用。
2. 💡 先前研究与新思路: 该工作基于 OpenClaw 风格的智能体框架、ReAct 推理模式、专用 GUI 智能体（CogAgent、UI-TARS、Aguvis）以及记忆/技能复用系统（Reflexion、Voyager、ReasoningBank）。其新思路是提出"Know Deeply, Act Perfectly"范式，并实例化为 Know–Route–Act–Reflect 四阶段闭环，将高层任务编排与底层 GUI 控制解耦，并具备自我演化的记忆与技能。
3. ❓ 问题: 论文解决了主流 OpenClaw 风格个人助手的两个核心瓶颈：(1) 跨平台 GUI 交互能力不足；(2) 缺乏内置的自演化机制，无法从执行经验中持续学习改进。
4. 🛠️ 方法: 论文提出了 KnowAct-GUIClaw 框架，核心方法包括：主智能体-执行器双层协作架构、以 POMDP 形式化建模的 GUI 控制、归因感知的持久化经验记忆、可自演化且具备状态验证的技能库、用于跨应用数据传输的类型化黑板机制，以及基于轨迹提炼与反思的技能与记忆进化机制。
5. 📊 结果与评估: KnowAct-GUIClaw 配合 Kimi-K2.6 在 MobileWorld GUI-Only 任务上取得 64.1% 的成功率（业界最优，超越 GPT-5.5 和 Seed-2.0-Pro），在 AndroidDaily 上达到 78.61%（resolved 设定），并展示出良好的模型间迁移能力（Kimi 提升 8.5%，Qwen3.5-35B 提升 16.2%），跨平台可用性方面 HarmonyOS 达 76.2%，Windows 达 70.0%。
