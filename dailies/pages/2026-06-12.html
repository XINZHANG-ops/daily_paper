Paper 1

EvoArena: Tracking Memory Evolution for Robust LLM Agents in Dynamic Environments

Published: 2026-06-10

Link: http://arxiv.org/pdf/2606.13681

1. 📘 Topic and Domain: This paper focuses on LLM agent evaluation and memory design in dynamic/evolving environments, spanning the domains of terminal workflows, software engineering, and user preference personalization.
2. 💡 Previous Research and New Ideas: It builds on prior agent benchmarks (e.g., WebArena, SWE-bench, GAIA, Terminal-Bench, PersonaMem) and existing memory systems (e.g., A-Mem, Memento-Skill) that typically consolidate memory into a single latest state. The new idea is modeling environment evolution as persistent versioned chains and treating memory updates as an append-only patch history.
3. ❓ Problem: The paper addresses the underexplored challenge that LLM agents are mostly evaluated on static snapshots and suffer from "state collapse" when memory overwrites earlier valid information, causing failures in tracking evolving environments.
4. 🛠️ Methods: The authors introduce EvoArena, a benchmark suite with three evolution regimes (Terminal-Bench-Evo, SWE-Chain-Evo, PersonaMem-Evo), and EvoMem, a patch-based memory paradigm that records non-additive memory updates as append-only patches (storing pre-update state, post-update state, rationale, and evidence) and retrieves relevant patches alongside the latest memory at inference time.
5. 📊 Results and Evaluation: EvoMem yields average gains of 1.5% on EvoArena (with larger chain-level gains of 3.7%) and improvements of 6.1% on GAIA and 4.8% on LoCoMo, while mechanistic analyses show it improves evidence capture and reduces regression rates in software evolution.
1. 📘 主题与领域: 论文聚焦于动态/演化环境下 LLM 智能体的评估与记忆设计，涵盖终端工作流、软件工程和用户偏好个性化等领域。
2. 💡 先前研究与新思路: 基于 WebArena、SWE-bench、GAIA、Terminal-Bench、PersonaMem 等智能体基准以及 A-Mem、Memento-Skill 等通常将记忆整合为单一最新状态的记忆系统；新思路是将环境演化建模为持久的版本链，并将记忆更新视为仅追加的补丁历史。
3. ❓ 问题: 论文解决了 LLM 智能体多在静态快照上评估、且因记忆覆盖早期有效信息而出现"状态坍缩"，导致在演化环境中跟踪能力不足这一被忽视的部署挑战。
4. 🛠️ 方法: 作者提出 EvoArena 基准套件（包含 Terminal-Bench-Evo、SWE-Chain-Evo、PersonaMem-Evo 三种演化场景）和 EvoMem 基于补丁的记忆范式，将非累加性记忆更新记录为仅追加的补丁（存储更新前后状态、原因和证据），并在推理时同时检索最新记忆与相关补丁。
5. 📊 结果与评估: EvoMem 在 EvoArena 上平均提升 1.5%（链级提升达 3.7%），在 GAIA 上提升 6.1%，在 LoCoMo 上提升 4.8%；机理分析表明它能改善证据捕获并降低软件演化中的回归率。

Paper 2

InterleaveThinker: Reinforcing Agentic Interleaved Generation

Published: 2026-06-10

Link: http://arxiv.org/pdf/2606.13679

1. 📘 Topic and Domain: This paper focuses on multimodal AI and computer vision, specifically addressing interleaved text-image generation for visual narratives, guidance, and embodied manipulation tasks.
2. 💡 Previous Research and New Ideas: Building on existing image generators (e.g., FLUX, Qwen-Image) and Unified Multimodal Models (UMMs) like Janus-Pro and BAGEL, the paper introduces a novel decoupled multi-agent Planner-Gen-Critic framework to overcome the limitations of entangled planning and evaluation in single models.
3. ❓ Problem: The paper addresses two critical issues: (1) the inability of existing image generators to natively perform interleaved generation due to image-only output architectures, and (2) the visual over-reliance and step-wise error accumulation that plague UMMs in long-horizon tasks.
4. 🛠️ Methods: The authors propose InterleaveThinker, a three-module pipeline (Planner for global upfront instruction generation, Generator as a frozen image generator, and Critic for step-wise evaluation and prompt refinement) trained via SFT cold-start and GRPO-based RL with a novel dual-reward strategy combining accuracy and step-wise rewards.
5. 📊 Results and Evaluation: InterleaveThinker achieves performance comparable to proprietary models like Nano Banana and GPT-5 on interleaved benchmarks (UEval, CoMM) and dramatically boosts the base FLUX.2-klein model on reasoning benchmarks, improving WISE from 0.47 to 0.73 and RISE from 13.3 to 28.9.
1. 📘 主题与领域: 本论文聚焦于多模态人工智能与计算机视觉领域，具体研究面向视觉叙事、引导和具身操作的图文交错生成任务。
2. 💡 先前研究与新思路: 基于现有图像生成器（如FLUX、Qwen-Image）和统一多模态模型（如Janus-Pro、BAGEL），本文提出了一种新颖的解耦多智能体Planner-Gen-Critic框架，以克服单一模型中规划与评估耦合的局限性。
3. ❓ 问题: 论文解决了两个关键问题：（1）现有图像生成器因仅支持图像输出而无法原生执行图文交错生成；（2）统一多模态模型在长程任务中存在的视觉过度依赖和逐步误差累积问题。
4. 🛠️ 方法: 作者提出了InterleaveThinker，这是一个三模块流水线（Planner用于全局前置指令生成、Generator作为冻结的图像生成器、Critic用于逐步评估和提示词优化），通过SFT冷启动和基于GRPO的强化学习进行训练，并采用结合准确度奖励和逐步奖励的新型双奖励策略。
5. 📊 结果与评估: InterleaveThinker在交错生成基准测试（UEval、CoMM）上达到了与Nano Banana和GPT-5等专有模型相当的性能，并在推理基准测试上显著提升了基础FLUX.2-klein模型，将WISE从0.47提升至0.73，RISE从13.3跃升至28.9。

Paper 3

SpatialClaw: Rethinking Action Interface for Agentic Spatial Reasoning

Published: 2026-06-10

Link: http://arxiv.org/pdf/2606.13673

1. 📘 Topic and Domain: The paper focuses on spatial reasoning for vision-language models (VLMs), specifically the design of action interfaces for tool-augmented spatial reasoning agents in 3D/4D visual understanding tasks.
2. 💡 Previous Research and New Ideas: The paper builds on prior work in tool-augmented visual agents (Visual Programming, ViperGPT, HuggingGPT) and spatial reasoning agents (SpaceTools, GCA, pySpatial, VADAR), as well as code action interfaces (CodeAct). Its new idea is to treat code as an "orchestration space" via a persistent multi-turn Python kernel, rather than as a one-shot program or a fixed tool-dispatch interface, enabling iterative composition and revision of perception results.
3. ❓ Problem: Existing spatial agents either commit to a complete analysis strategy before observing any intermediate result (single-pass code) or offer limited flexibility for composing operations tailored to each task (structured tool-calls), both of which are inadequate for open-ended, complex 3D/4D spatial reasoning.
4. 🛠️ Methods: The paper proposes SpatialClaw, a training-free framework featuring a persistent IPython kernel pre-loaded with input frames, perception tools (SAM3 for segmentation, Depth Anything 3 for reconstruction), and scientific libraries (NumPy, SciPy, Matplotlib). A VLM-backed agent writes one executable Python cell per step within a five-stage loop: Planning, Code Generation, Code Execution, Feedback Assembly, and Answer Submission, conditioned on prior text, variables, errors, and rendered visualizations.
5. 📊 Results and Evaluation: SpatialClaw achieves 59.9% average accuracy across 20 spatial reasoning benchmarks, outperforming the recent SpaceTools agent by +11.2 points, with consistent gains across six VLM backbones (Qwen3.5/3.6 and Gemma4 families, 26B–397B parameters) without any benchmark- or model-specific adaptation; the largest improvements appear on video/4D and multi-view reasoning tasks requiring chained geometric computation.
1. 📘 主题与领域: 论文聚焦于视觉语言模型（VLM）的空间推理，具体研究工具增强型空间推理智能体的动作接口设计，属于计算机视觉与多模态人工智能领域。
2. 💡 先前研究与新思路: 论文建立在工具增强视觉智能体（Visual Programming、ViperGPT、HuggingGPT）和空间推理智能体（SpaceTools、GCA、pySpatial、VADAR）以及代码动作接口（CodeAct）等先前研究之上。其新思路是将代码视为"编排空间"，通过持久化的多轮 Python 内核实现，而不仅仅是一次性程序或固定工具调度接口，从而支持感知结果的迭代组合与修正。
3. ❓ 问题: 现有空间智能体要么在观察任何中间结果之前就需提交完整分析策略（单次代码执行），要么在按需组合操作方面灵活性有限（结构化工具调用），二者均不足以应对开放式、复杂的 3D/4D 空间推理任务。
4. 🛠️ 方法: 论文提出 SpatialClaw，一种免训练框架，其核心是一个持久化的 IPython 内核，预先加载输入帧、感知工具（SAM3 分割、Depth Anything 3 三维重建）及科学计算库（NumPy、SciPy、Matplotlib）。VLM 驱动的智能体在五阶段循环（规划、代码生成、代码执行、反馈组装、答案提交）中逐步编写可执行 Python 单元，并基于先前的文本、变量、错误及可视化结果进行条件决策。
5. 📊 结果与评估: SpatialClaw 在 20 个空间推理基准上达到 59.9% 的平均准确率，较最近的 SpaceTools 智能体提升 +11.2 分；在六个 VLM 主干模型（Qwen3.5/3.6 和 Gemma4 系列，26B–397B 参数）上均取得稳定提升，且无需任何基准或模型特定的适配；最大改进出现在需要跨帧/跨视角链式几何计算的视频/4D 与多视角推理任务上。

Today's Reading Tips 今日阅读推荐

Start with Paper 1 (EvoArena/EvoMem) because the state-collapse problem it identifies is foundational and directly resonates with both other works: SpatialClaw's persistent IPython kernel and InterleaveThinker's Planner-Gen-Critic pipeline both implicitly rely on maintaining coherent, evolving state across multi-turn interactions—precisely the challenge EvoMem addresses via append-only patches. Reading it first will sharpen your lens for recognizing where memory versioning and evidence preservation could similarly improve spatial reasoning agents (Paper 3) and long-horizon multimodal generation (Paper 2).
建议先读论文1（EvoArena/EvoMem），因为它所揭示的"状态坍缩"问题是基础性的，与另外两篇工作直接相关：SpatialClaw 的持久化 IPython 内核和 InterleaveThinker 的 Planner-Gen-Critic 流水线都隐含地依赖在多轮交互中维护连贯、可演化的状态——这正是 EvoMem 通过仅追加补丁（append-only patches）所解决的核心挑战。优先阅读它有助于你在后续阅读中更敏锐地识别：版本化记忆与证据保存机制如何同样能提升空间推理智能体（论文3）和长链路多模态生成（论文2）的性能。
