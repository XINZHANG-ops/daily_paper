Paper 1

NatureBench: Can Coding Agents Match the Published SOTA of Nature-Family Papers?

Published: 2026-06-22

Link: http://arxiv.org/pdf/2606.24530

1. 📘 Topic and Domain: The paper introduces NatureBench, a cross-discipline benchmark in AI for Science and agent evaluation, designed to test whether AI coding agents can surpass the published state-of-the-art (SOTA) on real scientific problems derived from Nature-family journals.
2. 💡 Previous Research and New Ideas: It builds on prior paper-based reproduction benchmarks (e.g., PaperBench, CORE-Bench, ReplicationBench) and engineering-optimization benchmarks (e.g., MLE-bench, PostTrainBench, MLS-Bench), introducing the new idea of a discovery-oriented benchmark that uniquely combines paper-sourced tasks, genuine scientific domains, and SOTA-referenced evaluation.
3. ❓ Problem: It addresses the lack of a credible benchmark that evaluates whether AI coding agents can move beyond reproduction to genuine method discovery on natural science problems, overcoming the environment-fragmentation problem that has limited prior agent-on-research benchmarks.
4. 🛠️ Methods: The authors built NatureGym, an automated three-stage pipeline (Paper Filtering, Dataset Acquisition & Verification, Task Package Construction) that converts Nature-family papers into standardized containerized task packages with information firewalls, and evaluated 10 frontier agent configurations across 90 tasks spanning six scientific domains under a strict web-search-disabled protocol with SOTA-normalized relative gap scoring.
5. 📊 Results and Evaluation: The strongest agent (Claude Opus 4.7) surpasses SOTA on only 17.8% of tasks and matches it on 47.8%, with successes driven primarily by methodological translation into supervised prediction (45.5%) rather than scientific invention, and failures dominated by wrong method choice (45.1%) and insufficient compute budget (24.4%).
1. 📘 主题与领域: 该论文提出了NatureBench，一个跨学科的AI智能体评估基准，测试AI编程智能体能否在源自Nature系列期刊的真实科学问题上超越已发表的SOTA水平，跨越AI for Science与智能体评估领域。
2. 💡 先前研究与新思路: 该工作基于先前的论文复现基准（如PaperBench、CORE-Bench、ReplicationBench）和工程优化基准（如MLE-bench、PostTrainBench、MLS-Bench），提出了一种独特的发现导向基准，结合论文来源任务、真实科学领域与SOTA参考评估。
3. ❓ 问题: 该论文解决了现有基准的不足：要么仅关注论文方法的复现，要么仅针对Kaggle等工程优化问题，缺少能够评估AI编程智能体是否能在自然科学问题上实现真正方法发现的可信基准。
4. 🛠️ 方法: 作者构建了NatureGym，一个三阶段自动化流水线（论文筛选、数据获取与验证、任务包构建），将Nature系列论文转化为带有信息防火墙的标准化容器化任务包，并在禁用网络搜索的严格协议下，使用SOTA归一化相对差距评分，评估了10个前沿智能体配置在90个跨6个科学领域任务上的表现。
5. 📊 结果与评估: 最强的智能体（Claude Opus 4.7）仅在17.8%的任务上超越SOTA，在47.8%的任务上与之持平，成功主要通过将科学任务转化为有监督预测的方法学转换（占45.5%）而非真正的科学创新，失败主要由错误的方法选择（45.1%）和计算预算不足（24.4%）导致。

Paper 2

CLI-Universe: Towards Verifiable Task Synthesis Engine for Terminal Agents

Published: 2026-06-21

Link: http://arxiv.org/pdf/2606.22883

1. 📘 Topic and Domain: The paper addresses the data synthesis engine for LLM-based terminal agents, situated at the intersection of AI agents, synthetic data generation, and CLI task automation.
2. 💡 Previous Research and New Ideas: Building on prior terminal-agent work (e.g., Terminal-Bench, SWE-agent) and synthetic data pipelines (taxonomy-based and environment-harvesting approaches like SkillSynth and TerminalTraj), the paper proposes an "inside-out" synthesis engine that grounds tasks through structured capability specifications and evidence-guided deep research rather than retrofitting from surface artifacts.
3. ❓ Problem: It solves the scarcity of high-quality, executable training data for terminal agents, where existing pipelines scale task volume but produce ambiguous instructions, shallow execution paths, and brittle tests that yield weak learning signals.
4. 🛠️ Methods: A three-stage pipeline is used: (1) task blueprint construction via multi-dimensional capability taxonomy and evidence-guided deep research, (2) environment realization into Dockerized containers with materialized assets, and (3) multi-stage executable verification including rubric-gated test construction, hint-conditional filtering, and fail-to-pass checking.
5. 📊 Results and Evaluation: Fine-tuning Qwen3-32B on only 6,000 CLI-Universe trajectories achieves 33.4% on Terminal-Bench 2.0, setting a new SOTA among ≤32B open-source models and surpassing several models an order of magnitude larger, with strong generalization to BFCL v4 and VitaBench.
1. 📘 主题与领域: 论文聚焦于面向LLM终端代理的数据合成引擎,处于AI代理、合成数据生成与CLI任务自动化的交叉领域。
2. 💡 先前研究与新思路: 基于既有终端代理研究(如Terminal-Bench、SWE-agent)及合成数据流水线(基于分类法和环境采集的方法如SkillSynth、TerminalTraj),论文提出"由内而外"的合成引擎,通过结构化能力规范和证据引导的深度研究来锚定任务,避免从表面材料进行改造。
3. ❓ 问题: 论文解决了终端代理缺乏高质量、可执行训练数据的问题,现有流水线虽扩展了任务数量,但生成的指令模糊、执行路径浅、测试脆弱,导致学习信号微弱。
4. 🛠️ 方法: 论文采用三阶段流水线:(1)通过多维能力分类法和证据引导的深度研究构建任务蓝图;(2)将蓝图实现为包含具象化资产的Docker化环境;(3)多阶段可执行验证,包括基于评分标准的测试构建、提示条件过滤以及"失败-通过"校验。
5. 📊 结果与评估: 仅用6,000条CLI-Universe轨迹对Qwen3-32B进行微调,即在Terminal-Bench 2.0上取得33.4%,在≤32B开源模型中创下新SOTA,并超越多个大一个数量级的模型,同时在BFCL v4和VitaBench上表现出良好的泛化能力。

AOHP: An Open-Source OS-Level Agent Harness for Personalized, Efficient and Secure Interaction

Published: 2026-06-21

Link: http://arxiv.org/pdf/2606.23449

1. 📘 Topic and Domain: This paper belongs to the intersection of mobile operating systems and AI agents, proposing an open-source OS-level agent harness (AOHP) built on AOSP for personalized, efficient, and secure agent-mediated interaction on Android devices.
2. 💡 Previous Research and New Ideas: The paper builds on prior GUI-agent systems (e.g., AutoDroid, AppAgent, UI-TARS, OS-Copilot), Android automation frameworks (e.g., DroidBot, ScreenAI), and information-flow control systems (e.g., FIDES, TaintDroid); its new idea is to redesign the OS itself—treating agents as first-class OS actors—rather than only improving agent policies on top of a fixed, app-centric substrate.
3. ❓ Problem: The paper addresses the architectural mismatch between conventional app-centric operating systems and agent-mediated workflows, which causes execution overhead, inefficiency, and security risks because existing OSes cannot efficiently support cross-app agent tasks or properly track sensitive data propagation.
4. 🛠️ Methods: The paper designs and implements AOHP on AOSP with three core mechanisms—personalized service composition (task-level synthesized entrances), efficient agent interfaces (parallel background interaction, structured UI, native sandbox runtime, unified file shortcut, event stream abstraction), and secure information flow (source sanitization, vault-based trusted execution, and taint tracking)—and evaluates it using the OpenClaw agent on 30 self-crafted mobile benchmark tasks.
5. 📊 Results and Evaluation: Evaluation shows that AOHP raises task completion rate from 54.44% to 75.56% (+21.12%), reduces tool calls by 44.64%, duration by 44.21%, token consumption by 51.55%, and LLM requests by 47.62%, while security case studies on an annotated payment app confirm successful enforcement of sensitive-data sanitization, mediation, vault usage, taint propagation, and fail-closed behavior.
1. 📘 主题与领域: 本文属于移动操作系统与AI智能体交叉领域，提出基于AOSP构建的开源操作系统级智能体框架AOHP，用于在Android设备上实现个性化、高效且安全的智能体中介交互。
2. 💡 先前研究与新思路: 本文基于现有GUI智能体系统（如AutoDroid、AppAgent、UI-TARS、OS-Copilot）、Android自动化框架（如DroidBot、ScreenAI）以及信息流控制系统（如FIDES、TaintDroid）；其新思路在于重新设计操作系统本身，将智能体视为一等公民，而非仅在固定的应用中心化平台之上改进智能体策略。
3. ❓ 问题: 本文解决了传统以应用为中心的操作系统与智能体中介工作流之间的架构不匹配问题，该不匹配导致执行开销大、效率低下以及安全风险，因为现有OS无法高效支持跨应用智能体任务，也无法正确追踪敏感数据的传播。
4. 🛠️ 方法: 本文在AOSP上设计并实现了AOHP，包含三大核心机制——个性化服务组合（任务级生成入口）、高效智能体接口（并行后台交互、结构化UI、本地沙箱运行时、统一文件快捷方式、事件流抽象）以及安全信息流（源净化、基于保险库的受信执行和污点追踪），并使用OpenClaw智能体在30个自构移动基准任务上进行评估。
5. 📊 结果与评估: 评估结果显示，AOHP将任务完成率从54.44%提升至75.56%（+21.12%），工具调用减少44.64%，时长缩短44.21%，token消耗降低51.55%，LLM请求减少47.62%；同时在带注释的支付应用上的安全案例研究证实敏感数据净化、中介、保险库使用、污点传播和故障关闭行为均得到成功执行。

Today's Reading Tips 今日阅读推荐

I'd recommend reading the NatureBench paper first, as it provides the most thought-provoking findings: even Claude Opus 4.7 only surpasses SOTA on 17.8% of real scientific tasks, with failures dominated by wrong method choice—setting a sobering baseline that contextualizes the other two papers. The terminal-agent and AOHP papers both address agent capability and efficiency (training data synthesis, OS-level agent design) but from complementary angles: CLI-Universe focuses on building better training data for agents, while AOHP redesigns the OS substrate itself. Together, the three papers form a coherent picture of the current agent frontier—evaluation, training, and deployment infrastructure.
