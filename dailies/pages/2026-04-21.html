Paper 1

OpenGame: Open Agentic Coding for Games

Published: 2026-04-19

Link: http://arxiv.org/pdf/2604.18394

1. 📘 Topic and Domain: The paper presents OpenGame, an open-source agentic framework for end-to-end web game creation from natural language specifications, belonging to the domain of AI-assisted software engineering and autonomous code generation for interactive applications.
2. 💡 Previous Research and New Ideas: Based on prior work in agentic code generation (SWE-Bench, Cursor, qwen-code), game AI (SIMA, Genie), and code agents, the paper introduces novel contributions including Game Skill (combining Template Skill for stable scaffolding and Debug Skill for cumulative error repair), GameCoder-27B trained via three-stage pipeline, and OpenGame-Bench for dynamic playability evaluation.
3. ❓ Problem: The paper solves the challenge of generating fully playable, coherent web games from natural language, addressing three failure modes in existing LLMs: logical incoherence across game loops, engine-specific knowledge gaps, and cross-file inconsistencies that break multi-file projects.
4. 🛠️ Methods: The paper employs GameCoder-27B trained through continual pre-training, supervised fine-tuning, and execution-grounded reinforcement learning, combined with a six-phase autonomous workflow (classification, scaffolding, GDD generation, asset synthesis, implementation, verification) and Game Skill for evolving template libraries and living debugging protocols.
5. 📊 Results and Evaluation: OpenGame with Claude Sonnet 4.6 achieves state-of-the-art performance with Build Health=72.4, Visual Usability=67.2, and Intent Alignment=65.1, outperforming Cursor with Claude Sonnet 4.6 by 5.6, 5.8, and 6.2 points respectively across 150 diverse game prompts on OpenGame-Bench.
1. 📘 主题与领域: 本文提出OpenGame，一个用于从自然语言规范端到端创建网页游戏的开源智能体框架，属于人工智能辅助软件工程和交互式应用自主代码生成领域。
2. 💡 先前研究与新思路: 基于智能体代码生成（SWE-Bench、Cursor、qwen-code）、游戏AI（SIMA、Genie）和代码智能体的先前研究，本文引入创新贡献：Game Skill（结合用于稳定脚手架的模板技能和用于累积错误修复的调试技能）、通过三阶段训练的GameCoder-27B模型，以及用于动态可玩性评估的OpenGame-Bench。
3. ❓ 问题: 本文解决了从自然语言生成完全可玩、连贯的网页游戏的挑战，针对现有LLM的三个失败模式：游戏循环中的逻辑不一致、引擎特定知识差距，以及破坏多文件项目的跨文件不一致问题。
4. 🛠️ 方法: 本文采用通过持续预训练、监督微调和基于执行的强化学习训练的GameCoder-27B，结合六阶段自主工作流（分类、脚手架、GDD生成、资产合成、实现、验证）以及Game Skill（用于不断演进的模板库和活体调试协议）。
5. 📊 结果与评估: 使用Claude Sonnet 4.6的OpenGame以构建健康度72.4、视觉可用性67.2和意图对齐度65.1取得最先进性能，在OpenGame-Bench的150个多样化游戏提示上分别超越使用Claude Sonnet 4.6的Cursor 5.6、5.8和6.2分。

MultiWorld: Scalable Multi-Agent Multi-View Video World Models

Published: 2026-04-19

Link: http://arxiv.org/pdf/2604.18564

1. 📘 Topic and Domain: Multi-agent multi-view video world models for simulating environmental dynamics in response to actions by multiple agents, spanning video game and robotic manipulation domains.
2. 💡 Previous Research and New Ideas: Based on single-agent video world models using diffusion Transformers; introduces Multi-Agent Condition Module (MACM) and Global State Encoder (GSE) to enable multi-agent controllability and multi-view consistency while maintaining scalability.
3. ❓ Problem: The paper solves three key challenges: (1) multi-agent controllability where actions must be correctly associated with specific agents, (2) multi-view consistency where different viewpoints must produce coherent observations, and (3) framework scalability for variable agent and camera counts without architectural changes.
4. 🛠️ Methods: MACM uses Agent Identity Embedding (via rotary position embeddings) and Adaptive Action Weighting to handle agent distinction and prioritize active agents; GSE leverages a pretrained VGGT model to extract 3D-aware global environment states from multiple views for consistent multi-view generation.
5. 📊 Results and Evaluation: MultiWorld outperforms all baselines on video fidelity (FVD: 179 vs 245 for standard), action-following (89.8% vs 88.4%), and multi-view consistency (RPE: 0.67 vs 0.75), validated on a 100-hour ItTakesTwo game dataset and multi-robot RoboFactory manipulation tasks.
1. 📘 主题与领域: 多智能体多视角视频世界模型，用于模拟多个智能体响应动作时的环境动态，涵盖视频游戏和机器人操作领域。
2. 💡 先前研究与新思路: 基于单智能体视频世界模型（使用扩散Transformer）；创新性地引入多智能体条件模块（MACM）和全局状态编码器（GSE），实现多智能体可控性和多视角一致性，同时保持框架的可扩展性。
3. ❓ 问题: 论文解决了三个关键挑战：(1) 多智能体可控性，动作必须与特定智能体正确关联；(2) 多视角一致性，不同视角必须产生一致的观测结果；(3) 框架可扩展性，无需架构改变即可适应可变的智能体和相机数量。
4. 🛠️ 方法: MACM使用智能体身份嵌入（通过旋转位置编码）和自适应动作权重来处理智能体区分并优先处理活跃智能体；GSE利用预训练的VGGT模型从多个视角提取3D感知的全局环境状态，以实现一致的多视角生成。
5. 📊 结果与评估: MultiWorld在所有基线上均表现优异，视频保真度（FVD: 179 vs 标准方法的245）、动作跟随（89.8% vs 88.4%）和多视角一致性（RPE: 0.67 vs 0.75）均显著提升，在100小时的ItTakesTwo游戏数据集和多机器人RoboFactory操作任务上得到验证。

Paper 3

When Can LLMs Learn to Reason with Weak Supervision?

Published: 2026-04-19

Link: http://arxiv.org/pdf/2604.18574

1. 📘 Topic and Domain: Reinforcement Learning with Verifiable Rewards (RLVR) for large language model reasoning, studying generalization under weak supervision across model families (Qwen, Llama) and reasoning domains (MATH, SCIENCE, GRAPH).
2. 💡 Previous Research and New Ideas: Based on prior RLVR research, this paper introduces: (1) training reward saturation dynamics as the key factor governing generalization, (2) reasoning faithfulness (not output diversity) as the predictor of which models generalize vs. memorize, and (3) that SFT on explicit reasoning traces combined with continual pre-training recovers generalization under weak supervision.
3. ❓ Problem: When can RLVR generalize under weak supervision (scarce data, noisy rewards, self-supervised proxy rewards), and what determines success or failure across different model families and domains?
4. 🛠️ Methods: Systematic empirical study using GRPO algorithm across diverse models and domains; analysis of training reward saturation dynamics; measurement of reasoning faithfulness via LLM-as-judge; pre-RL interventions combining continual pre-training (CPT) on math data and SFT with explicit reasoning traces.
5. 📊 Results and Evaluation: Models with strong domain-aligned pretraining (Qwen on MATH/SCIENCE) generalize under all three weak supervision settings; those without (Llama) fail due to rapid saturation and memorization. Reasoning faithfulness, not output diversity, distinguishes successful models. CPT + Thinking SFT enables Llama3.2-3B-Base to generalize where it previously failed. Qwen tolerates up to 70% reward corruption; Llama degrades at 50%.
1. 📘 主题与领域: 大语言模型的强化学习可验证奖励（RLVR）推理，研究在模型家族（Qwen、Llama）和推理领域（MATH、SCIENCE、GRAPH）中弱监督下的泛化能力。
2. 💡 先前研究与新思路: 基于先前的RLVR研究，本文提出：(1) 训练奖励饱和动力学是泛化的关键因素；(2) 推理忠诚度（而非输出多样性）是模型泛化与记忆的预测因子；(3) 带有显式推理轨迹的SFT结合持续预训练可恢复弱监督下的泛化能力。
3. ❓ 问题: 在弱监督（数据稀缺、有噪声奖励、自监督代理奖励）条件下，RLVR何时能泛化，什么因素决定不同模型家族和领域的成功或失败？
4. 🛠️ 方法: 使用GRPO算法在不同模型和领域进行系统性实证研究；分析训练奖励饱和动力学；通过LLM评判器测量推理忠诚度；结合数学数据持续预训练（CPT）和带显式推理轨迹的SFT进行预RL干预。
5. 📊 结果与评估: 具有强领域对齐预训练的模型（Qwen在MATH/SCIENCE上）能在三种弱监督设置下泛化；缺少此类预训练的模型（Llama）因快速饱和和记忆而失败。推理忠诚度而非输出多样性区分成功模型。CPT + Thinking SFT使Llama3.2-3B-Base能够在之前失败的任务上泛化。Qwen能容忍高达70%的奖励损坏；Llama在50%时性能下降。

Today's Reading Tips 今日阅读推荐

Start with the RLVR paper (Paper 3) as it provides foundational insights into LLM training dynamics and generalization under weak supervision, which can help you understand the training methodology used in OpenGame's GameCoder (Paper 1). Papers 1 and 2 share a game domain focus but approach it from different angles—OpenGame creates games while MultiWorld simulates game environments—making them complementary reads for a comprehensive view of game AI.
建议先读RLVR论文（第三篇），因为它提供了LLM训练动态和弱监督下泛化的基础性见解，有助于理解OpenGame中GameCoder的训练方法（第一篇）。第一篇和第二篇都以游戏领域为重点，但切入点不同——OpenGame创作游戏，MultiWorld模拟游戏环境——可作为游戏AI的互补阅读材料。
