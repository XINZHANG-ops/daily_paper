δ-mem: Efficient Online Memory for Large Language Models

Published: 2026-05-11

Link: http://arxiv.org/pdf/2605.12357

1. 📘 Topic and Domain: The paper proposes δ-mem, a lightweight online memory mechanism for large language models, within the domain of natural language processing and AI agent systems. 2. **💡 Previous Research and New Ideas:** It builds upon prior textual, outside-channel, and parametric memory paradigms, and introduces a compact, dynamically updated online state of associative memory using delta-rule learning that directly steers a frozen backbone’s attention computation via low-rank corrections. 3. **❓ Problem:** It solves the problem that simply expanding context windows is computationally expensive and suffers from context degradation, while existing memory mechanisms fail to simultaneously maintain a compact, dynamically evolving memory state and tightly couple it with the backbone’s internal reasoning. 4. **🛠️ Methods:** The method compresses historical information into a fixed-size matrix state updated by gated delta-rule learning, reads associative memory signals from this state to generate low-rank corrections to the backbone’s query and output attention branches during generation, and explores token-level, segment-level, and multi-state writing strategies. 5. **📊 Results and Evaluation:** Evaluation on memory-heavy benchmarks (MemoryAgentBench, LoCoMo, HotpotQA) and general tasks shows that an 8×8 online state improves the average score to 1.10× that of the frozen backbone and 1.15× that of the strongest baseline, with gains up to 1.31× on memory-heavy tasks, while preserving general capabilities and introducing minimal parameter overhead.

RubricEM: Meta-RL with Rubric-guided Policy Decomposition beyond Verifiable Rewards

Published: 2026-05-10

Link: http://arxiv.org/pdf/2605.10899

1. 📘 Topic and Domain: The paper presents RubricEM, a reinforcement learning framework in the AI/ML domain for training deep research agents to autonomously plan, search, evaluate, and synthesize long-form reports beyond verifiable rewards. 2. **💡 Previous Research and New Ideas:** Building on prior work such as DR Tulu that uses rubrics only for final-answer evaluation, the novel idea is to treat rubrics as a shared interface structuring stagewise policy execution, process-level judge feedback, and reusable agent memory via a reflection meta-policy. 3. **❓ Problem:** It solves the problem of how to train deep research agents with reinforcement learning when tasks lack verifiable ground truth, trajectories are long-horizon and tool-augmented, and feedback is sparse and delayed without reusable experience. 4. **🛠️ Methods:** The method uses a rubric-guided four-stage reasoning scaffold (Plan, Research, Review, Answer), Stage-Structured GRPO for dense stagewise credit assignment, and a jointly trained shared-backbone reflection meta-policy with an asynchronous pipeline to distill judged trajectories into a reusable rubric bank. 5. **📊 Results and Evaluation:** RubricEM-8B achieves state-of-the-art results among open models across four long-form benchmarks, approaching proprietary systems while using fewer RL steps than prior work, with strong transfer to short-form tasks and ablations confirming each component's contribution.
