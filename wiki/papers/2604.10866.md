# OccuBench: Evaluating AI Agents on Real-World Professional Tasks via Language World Models

**Published**: 2026-04-12 | **Added**: 2026-04-20 | **arXiv**: [2604.10866](http://arxiv.org/pdf/2604.10866)

## Summary

OCCUBENCH addresses a fundamental evaluation gap: the professional domains where AI agents would deliver the most value (healthcare, finance, legal, manufacturing) are precisely the domains where no benchmarks exist, because they require access to enterprise systems with no public APIs. The paper introduces Language Environment Simulators (LES) — LLM-driven simulators that can model any professional domain's tool-response behavior given only a configuration (system prompt, tool schema, initial state, state description). This transforms environment construction from an engineering problem into a configuration problem, extending benchmark coverage from "domains with public environments" to "any domain an LLM can understand." The benchmark covers 100 scenarios across 10 industry categories and 65 specialized domains, evaluating both task completion and environmental robustness under fault injection.

## Key Contributions

- **Language Environment Simulator (LES)**: LLM-driven environment simulation using only configuration files — no public APIs or enterprise systems required
- **100 professional scenarios** across 10 industries: Business, Technology, Industrial, Transportation, Commerce, Education, Healthcare, Public Service, Agriculture, Science
- **Fault injection robustness evaluation**: Three fault types (E0 clean, E1 explicit errors like HTTP 500s, E2 implicit degradation like truncated data, E3 mixed)
- **Cross-industry capability profiling**: Reveals that each model has distinct occupational specializations invisible to single-domain benchmarks

## Method

**LES Architecture**: The Language Environment Simulator is formalized as a function f_θ(s_t, a_t; c) where configuration c = (system prompt, tool schema, initial state, state description). The LLM maintains latent state through its context window, generating tool responses conditioned on the configuration and conversation history.

**Multi-Agent Synthesis Pipeline**: Uses Gemini-3-Flash-Preview as the LES to generate environment configurations, task instructions, tool definitions, solution plans, and verification rubrics. Each task is executed multiple times to verify solvability and calibrate difficulty.

**Evaluation Dimensions**:
1. **Task Completion**: Multi-step decision-making across industries
2. **Environmental Robustness**: Performance under fault injection (explicit errors, implicit degradation, mixed faults)

**Fault Types**:
- E0: Clean baseline
- E1: Explicit faults (HTTP 500, TimeoutError, ConnectionRefused) — clear error signals, prompts retry
- E2: Implicit faults (truncated data, missing fields) — no error signal, requires independent quality detection
- E3: Mixed explicit/implicit faults

## Results

**Key Finding 1: No single model dominates all industries**
| Model | Best Industries | Weakest Industries |
|-------|-----------------|---------------------|
| GPT-5.2 (79.6%) | Agriculture 84%, Business 86%, Science 94% | Commerce 67% |
| Gemini 3.1 Pro (72.3%) | Education 84%, Science 81% | Healthcare 62% |
| Claude Opus 4.6 (71.5%) | Transportation 77%, Business 78% | Commerce 53% |
| Qwen 3.5 Plus (69.9%) | Healthcare 81%, Commerce 81% | Education 56% |

**Key Finding 2: Implicit faults are harder than explicit faults**
- E0: 67.5% average completion rate
- E1: 62.6% (explicit errors with clear signals)
- E2: 53.4% (silent data degradation) — **largest drop**
- E3: 54.4% (mixed)

The counter-intuitive result: implicit faults lack error signals, so agents cannot know data is degraded and re-query. Claude Opus 4.6 drops 17.6 points under E2 (71.5% → 53.9%).

**Key Finding 3: Scaling consistently improves performance**
- GPT-5.2 improves 27.5 points from none to xhigh reasoning effort (54.7% → 82.2%)
- Within-family gaps: Gemini Pro vs Flash-Lite (11.0%), Qwen Plus vs Flash (10.2%)

**Key Finding 4: Strong agents are not necessarily strong simulators**
- GPT-5.2 ranks #1 as an agent (79.6%) but produces worst simulation quality
- Under GPT-5.2 simulator, all agents average only 29.3% vs 67.9% under Gemini Flash
- Three failure modes: state fabrication, entity omission, rule invention
- Qwen 3.5 Plus simulator achieves 85.7% pairwise ranking agreement with Gemini Flash
