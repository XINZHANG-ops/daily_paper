# DataFlex: A Unified Framework for Data-Centric Dynamic Training of Large Language Models

**Published**: 2026-03-27 | **Added**: 2026-04-06 | **arXiv**: [2603.26164](http://arxiv.org/pdf/2603.26164)

## Summary

DataFlex addresses the fragmentation problem in data-centric LLM training. While dynamic data selection, mixture optimization, and sample reweighting have each shown promise individually, they exist in isolated codebases with incompatible interfaces. DataFlex unifies these three paradigms into a single framework built on LLaMA-Factory, enabling researchers to plug in different algorithms without changing the training pipeline. The key insight is that all three paradigms share the same underlying operations—embedding extraction, model inference, gradient computation—so unifying them yields both engineering efficiency and methodological comparability.

## Key Contributions

- First unified framework integrating dynamic sample selection, domain mixture adjustment, and sample reweighting
- Modular trainer abstractions (Select Trainer, Mix Trainer, Weight Trainer) that are algorithm-agnostic
- Full compatibility with LLaMA-Factory workflow and DeepSpeed ZeRO-3 for large-scale training
- Comprehensive benchmark across 7 data selection algorithms, 4 mixture methods, and 5 reweighting approaches

## Method

DataFlex replaces LLaMA-Factory's training layer with three unified trainer types. **Select Trainer** handles dynamic sample selection (which data to include), **Mix Trainer** handles domain mixture optimization (how to balance data sources), and **Weight Trainer** handles sample reweighting (how much to emphasize each sample). Each trainer accepts pluggable algorithm components—for example, LESS for selection, DoReMi for mixture, ODM for reweighting. The framework normalizes operations like embedding extraction and gradient computation that previously differed across implementations.

## Results

Dynamic methods consistently outperform static full-data training on MMLU accuracy. LESS achieves 5.8 percentage points improvement on Mistral-7B over static baselines. DoReMi and ODM improve both MMLU and perplexity on SlimPajama. Runtime is reduced by 3-7% compared to original standalone implementations, with 57% speedup on 8 GPUs for LESS due to shared infrastructure.
