# Claw-Eval: Toward Trustworthy Evaluation of Autonomous Agents

**Published**: 2026-04-06 | **Added**: 2026-04-08 | **arXiv**: [2604.06132](http://arxiv.org/pdf/2604.06132)

## Summary

Claw-Eval addresses three critical gaps in existing agent benchmarks: trajectory-opaque grading that only checks final outputs, underspecified safety and robustness evaluation, and narrow modality coverage. The framework introduces full-trajectory auditing with three independent evidence channels (execution traces, audit logs, environment snapshots), integrated multi-dimensional scoring (Completion, Safety, Robustness), and unified cross-modal coverage across 300 tasks spanning General service orchestration, Multimodal perception/generation, and Multi-turn professional dialogue.

The benchmark evaluates 14 frontier models and reveals striking findings: trajectory-opaque evaluation misses 44% of safety violations and 13% of robustness failures; controlled error injection primarily degrades consistency (Pass^3 drops up to 24%) while Pass@3 remains stable; multimodal performance varies sharply with video tasks substantially harder than document/image or code tasks; and no single model dominates across all modalities. Question quality (r=0.87) predicts multi-turn dialogue success far better than conversation length (r=0.07).

## Key Contributions

- **Full-Trajectory Auditing**: Three independent evidence channels enable verification of what the agent actually did rather than what it reported. Evidence channels are invisible to the agent during execution, ensuring genuine problem-solving behavior
- **Multi-Dimensional Scoring**: Completion (weighted rubric items), Safety (multiplicative gate), and Robustness (controlled error injection) evaluated together
- **Cross-Modal Coverage**: 300 tasks across 9 categories and 3 groups under a unified task schema and execution pipeline
- **2,159 Fine-Grained Rubric Items**: Mean 7.2 per task, providing evaluation surface substantially richer than task count alone
- **Three Complementary Metrics**: Average Score (overall capability), Pass@3 (capability ceiling), Pass^3 (reliability floor)

## Method

### Three-Phase Execution Lifecycle

**Phase 1 - Setup**: Task definition is read; fresh Docker sandbox container is provisioned with declared workspace files (datasets, documents, starter code, media assets). Mock services (CRM, email, scheduling, knowledge bases) are deployed outside the sandbox, accessible only through designated tool-call interfaces. Audit logs begin accumulating silently.

**Phase 2 - Execution**: Agent works within the isolated container, iteratively reasoning and taking actions. Two capability layers:
- **System Layer**: 11 built-in tools (Bash, Read, Write, Edit, Glob, Grep, Browser, Screenshot, WebSearch, WebFetch, ReadMedia, Download)
- **Service Layer**: Task-specific custom tools interfacing with mock services

Complete agentic context is recorded in a structured execution trace, maintained outside the sandbox and invisible to the agent.

**Phase 3 - Judge**: Upon agent termination, grading artifacts are injected for the first time. The pipeline assembles three independent evidence lines:
- **Execution Trace**: Complete agentic context including all tool calls
- **Audit Logs**: Every API request actually received by mock services with full parameters
- **Environment Snapshot**: Physical end-state of produced artifacts, files, verification outputs

This triangulation of what the agent said, what services observed, and what the environment contains transforms evaluation from trusting self-report to verifying actual behavior.

### Scoring Protocol

**Multi-Dimensional Structure**:
```
score = s_safety × (α · s_completion + β · s_robustness)
```
where α=0.8, β=0.2 (completion is primary, robustness is secondary).

**Completion**: Aggregated from task-specific rubric weights. Rubric items are either deterministic checks (file exists, API invoked with correct parameters) or LLM-judged assessments (text quality, reasoning coherence).

**Safety (Multiplicative Gate)**: A policy violation pulls entire score toward zero regardless of completion. Embedded within normal workflow tasks—not isolated red-teaming. Tests whether agents respect constraints under genuine task-completion pressure.

**Robustness**: Measured through controlled error injection. Mock services return errors at configurable rate (HTTP 429 rate-limit, HTTP 500 internal error, or 2-4s latency spike). Score captures breadth of recovery:
```
s_robustness = |Trecovered| / |Terrored| if |Terrored| > 0, else 1
```

### Evaluation Metrics

For N tasks, k=3 independent trials, pass threshold τ:

- **Average Score**: Mean task score across all runs, measuring overall capability level
- **Pass@3**: Fraction of tasks passed at least once, measuring capability ceiling
- **Pass^3**: Fraction of tasks passed on every trial, measuring reliability floor

The gap between Pass@3 and Pass^3 exposes tasks the model can sometimes complete but cannot reliably reproduce.

## Results

### Main Results: General & Multi-turn Tasks

| Model | General Score | General Pass@3 | General Pass^3 | Multi-turn Score | Multi-turn Pass@3 | Multi-turn Pass^3 |
|-------|--------------|----------------|----------------|------------------|--------------------|-------------------|
| Claude Opus 4.6 | 80.6 | 80.8 | 70.8 | 79.6 | 89.5 | 68.4 |
| Claude Sonnet 4.6 | 81.3 | 81.4 | 68.3 | 81.9 | 89.5 | 65.8 |
| GPT 5.4 | 78.3 | 75.8 | 60.2 | 79.0 | 89.5 | 60.5 |
| Gemini 3.1 Pro | 76.6 | 80.8 | 55.9 | 80.2 | 92.1 | 65.8 |
| Qwen 3.5 397A17B | 73.8 | 70.8 | 57.8 | 75.6 | 76.3 | 52.6 |

### Trajectory-Opaque Evaluation is Unreliable

Comparison of vanilla LLM judge (Gemini-3-Flash) vs hybrid grading pipeline across 5 models, 2000+ traces:

- **Safety Violations**: Hybrid pipeline detects 27 task-level violations; vanilla judge misses 12 (44% miss rate). Hybrid uses deterministic string matching on tool-call parameters; LLM cannot reliably execute substring matching mentally and sometimes rationalizes agent behavior.

- **Robustness Issues**: Hybrid pipeline detects 118 task-level issues; vanilla judge misses 15 (13% miss rate). Lower rate reflects visible error codes in conversation; safety violations require parameter-level inspection.

### Error Injection: Consistency vs Peak Capability

Pass@3 remains largely stable across error injection rates 0.0-0.6:
- Claude-Opus-4.6: drops only 3.7%
- GLM-5-Turbo: actually rises 1.2%

Pass^3 drops sharply:
- Gemini-3.1-Pro: -24.2% (most severe degradation)
- Claude-Opus-4.6: -14.3%
- GLM-5-Turbo: -12.4%

The Pass@3-Pass^3 gap widens monotonically for every model, quantifying the capability-reliability divide.

### Multi-turn Dialogue: Quality Over Quantity

Question precision (clarification targeting + trajectory logicality) explains 76% of Pass^3 variance (r=0.87). Round count shows near-zero correlation (r=0.07, R^2<0.01). Critical information is deliberately withheld behind progressive revelation; the quality of the questioning strategy determines how much context the model receives.

### Multimodal: Domain-Specific Performance

| Domain | Best Model | Best Pass^3 | Avg Pass^3 | Consistency Ratio |
|--------|-----------|-------------|------------|-------------------|
| Video | Claude Opus/Sonnet 4.6 | 15.4% | 10.7% | r=0.37 |
| Doc & Image | GPT 5.4 | 54.5% | 32.3% | r=0.53 |
| Code | MiMo V2 Omni | 33.3% | 23.9% | r=0.48 |

No single model dominates all domains. Video understanding involving frame sampling, temporal reasoning, and OCR remains the hardest challenge for current agents.

## Limitations & Future Work

The benchmark focuses on English-language tasks; multilingual coverage remains future work. Error injection patterns could expand to diverse failure modes beyond rate limits, server errors, and latency spikes. The framework is designed for extensibility—covering new domains requires only a task definition and grader script.
