# OpenGame: Open Agentic Coding for Games

**Published**: 2026-04-19 | **Added**: 2026-04-21 | **arXiv**: [2604.18394](http://arxiv.org/pdf/2604.18394)

## Summary

OpenGame is an open-source agentic framework for end-to-end web game creation from natural language specifications. The key insight is that general-purpose code agents fail at game generation due to three structural failure modes: logical incoherence across game loops, engine-specific knowledge gaps, and cross-file inconsistencies that break multi-file projects. OpenGame addresses these through Game Skill—a reusable capability combining Template Skill (stable project scaffolding via evolving template library) and Debug Skill (cumulative error repair via living debugging protocol). The framework uses GameCoder-27B, a domain-specialized code model trained through CPT, SFT, and execution-grounded RL, and evaluates via OpenGame-Bench which measures Build Health, Visual Usability, and Intent Alignment through headless browser execution.

## Key Contributions

- First open-source agentic framework for end-to-end playable web game generation
- Game Skill combining Template Skill (evolving template library) and Debug Skill (living debugging protocol)
- GameCoder-27B trained via three-stage pipeline: CPT, SFT, execution-grounded RL
- OpenGame-Bench evaluation pipeline with dynamic playability assessment

## Method

GameCoder-27B is built on Qwen3.5-27B backbone with three-stage training:
1. **Continual Pre-Training**: Phaser/JS repositories, documentation, tutorials
2. **Supervised Fine-Tuning**: GPT-5.1 curated prompts + MiniMax-2.5 solutions
3. **Reinforcement Learning**: Component-level logic with unit test rewards

Six-phase autonomous workflow:
1. **Classification**: Physics-first logic categorizes game archetype
2. **Scaffolding**: Template library instantiation
3. **GDD Generation**: Technical game design document
4. **Asset Synthesis**: Multimodal generation for visuals/audio
5. **Implementation**: Hook-driven coding with template method pattern
6. **Verification**: Debug protocol + iterative self-correction

## Results

OpenGame with Claude Sonnet 4.6 achieves SOTA on OpenGame-Bench (150 diverse game prompts):
- Build Health: 72.4 (vs Cursor 66.8)
- Visual Usability: 67.2 (vs Cursor 61.4)
- Intent Alignment: 65.1 (vs Cursor 58.9)

GameCoder-27B alone outperforms all direct LLMs on Build Health (63.9) and Intent Alignment (54.1).

## Limitations & Future Work

- Approximately 34.9% of weighted mechanical requirements remain unsatisfied
- Strategy and Puzzle genres degrade more than physics-centric genres
- Silent logic errors (inventory desynchronization) are harder to detect than compiler errors
