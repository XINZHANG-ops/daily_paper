## Summary

ClawBench introduces a benchmark of 153 everyday online tasks spanning 15 life categories across 144 live platforms, evaluating AI agents on write-heavy, state-changing web interactions that people accomplish regularly—booking flights, ordering groceries, submitting job applications. The key insight is that existing benchmarks evaluate agents in offline sandboxes with static HTML and fixed DOM structures, or restrict evaluation to read-only information retrieval, leaving the category most directly relevant to people's daily lives unevaluated. This controlled setting removes the very complexities that make real-world web interaction difficult: cookie consent popups, dynamic JavaScript rendering, complex multi-step interaction, and anti-bot defenses.

The benchmark addresses safety concerns by implementing a lightweight Chrome extension with CDP-based instrumentation that intercepts only the final submission request—the single HTTP call that would commit an irreversible transaction. This enables safe evaluation on production websites without real-world side effects while preserving full ecological validity. The evaluation pipeline captures five synchronized layers of behavioral data: session replay via Xvfb and FFmpeg, per-step action screenshots, HTTP traffic logs, agent messages (reasoning traces and tool calls), and low-level browser actions. Human annotators complete every task under the same setup to produce ground-truth trajectories, enabling an Agentic Evaluator implemented via Claude Code sub-agent to compare agent trajectories against human references and produce binary verdicts with structured justifications grounded in step-level evidence.

The results reveal a substantial gap between benchmark performance and real-world capability: Claude Sonnet 4.6 achieves 65-75% on OSWorld and WebArena but only 33.3% on ClawBench; GPT-5.4 achieves similar scores on traditional benchmarks but only 6.5% on ClawBench. This saturation of existing benchmarks while ClawBench remains challenging indicates the benchmark provides a meaningfully different evaluation setting.

## Key Contributions

- **153 Real-World Tasks on 144 Live Platforms**: Comprehensive coverage of everyday online tasks across Daily Life (21), Academia (5), Job Search (8), Travel (13), Shopping (16), Social (8), Entertainment (15), with write-heavy state-changing workflows that existing benchmarks do not cover.

- **Safe Live-Website Evaluation**: Lightweight Chrome extension and CDP server that intercepts only the final submission request, enabling evaluation on production websites without real-world side effects—no orders placed, applications submitted, or critical server-side state modified.

- **Five-Layer Recording Infrastructure**: Session replay, action screenshots, HTTP traffic, agent messages, and browser actions captured for every run, enabling both automated evaluation and deep post-hoc failure diagnosis traceable to exact steps.

- **Agentic Evaluator**: Claude Code sub-agent that compares agent trajectory against human reference trajectory across five evidence layers under a fixed evaluation rubric, producing binary verdicts with structured justifications grounded in schema-level checks over request payloads.

- **Comprehensive Model Comparison**: Evaluation of 7 frontier models (Claude Sonnet 4.6, GPT-5.4, Gemini 3.1 Flash Lite, Claude Haiku 4.5, Gemini 3 Flash, GLM-5, Kimi K2.5) revealing that strong performance on existing benchmarks does not transfer to real-world everyday web competence.

## Method

**Task Design and Collection**: Tasks are constructed from three elements: natural-language user instruction, starting URL, and terminal submission target specified at HTTP-request level. Human annotators survey representative platforms across life categories, instantiate realistic user goals, and complete each task end-to-end within the recording framework. A rigorous multi-stage filtering pipeline removes tasks requiring paid subscriptions, geographically restricted services, or offline websites. A distinguishing aspect is the manual annotation of interception signals—the specific HTTP endpoint, request method, and payload schema identifying the dangerous submission—by human experts inspecting browser network traffic during ground-truth execution.

**Interception Mechanism**: The instrumentation server connects to the browser through CDP to monitor all outgoing HTTP requests against human-annotated interception specifications. When an agent's action triggers a request matching the declared URL pattern and HTTP method, the system captures the full request body, blocks the request before it reaches the server, and logs the captured payload. All other requests—page loads, AJAX calls, image fetches, analytics—pass through unmodified. Validation study over all 153 tasks shows 100% correct interception with zero false positives on navigation traffic.

**Five-Layer Recording**: (1) Session Recording: Chromium on Xvfb virtual display monitored by FFmpeg, producing full-session video. (2) Action Screenshots: Per-step screenshot after each browser action (click, type, scroll) providing chronological visual record. (3) HTTP Traffic: All requests logged via CDP including bodies, payloads, timing. (4) Agent Messages: Full chain of reasoning traces, tool calls, intermediate outputs in structured JSON. (5) Browser Actions: Low-level events—mouse clicks with coordinates, keystrokes, scroll offsets, tab switches, navigation events—captured via Chrome extension.

**Agentic Evaluator**: Given task instruction q(t), agent trajectory T_a, and human reference trajectory T_h, the evaluator performs explicit alignment between two executions: identifies corresponding steps, detects divergences, checks whether required fields and actions are correct, determines whether terminal state matches reference. Key rules include: must use provided personal info correctly; must complete forms and submit; interceptor/phone verification blocks count as PASS if prior actions correct; CAPTCHA must be attempted or FAIL.

## Limitations & Future Work

The paper focuses on browser-based web agents controlling Chromium, leaving other agent types (mobile, desktop, API-based) as future work. The 153-task dataset, while comprehensive, represents a snapshot of everyday web tasks—continued expansion and maintenance requires community contribution. The five-layer recording infrastructure and agentic evaluator require significant computational resources, potentially limiting adoption. Future directions include expanding task categories, adding multilingual and mobile web evaluation, and investigating whether the gap between benchmark performance and real-world competence can be bridged through targeted agent improvements or architectural changes.
