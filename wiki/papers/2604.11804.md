# OmniShow: Unifying Multimodal Conditions for Human-Object Interaction Video Generation

**Published**: 2026-04-12 | **Added**: 2026-04-14 | **arXiv**: [2604.11804](http://arxiv.org/pdf/2604.11804)

## Summary

OmniShow is an end-to-end framework for Human-Object Interaction Video Generation (HOIVG) that unifies four distinct multimodal conditions: text, reference images, audio, and pose. Built on Waver 1.0 (12B MMDiT-based latent diffusion model), it introduces Unified Channel-wise Conditioning for efficient multimodal injection without disrupting generative priors, Gated Local-Context Attention for precise audio-visual synchronization, and a Decoupled-Then-Joint Training strategy to leverage heterogeneous sub-task datasets through model merging. At 12.3B parameters, it is the smallest 10B-scale model achieving state-of-the-art or highly competitive performance across R2V, RA2V, and RP2V settings, demonstrating superior reference consistency, audio-visual synchronization, and video quality on the HOIVG-Bench benchmark (135 curated samples).

## Key Contributions

- First unified framework for HOIVG supporting all four conditions (text, reference images, audio, pose)
- Unified Channel-wise Conditioning: augments noisy video tokens with pseudo-frame tokens and uses channel concatenation for seamless injection
- Gated Local-Context Attention: audio context packing + masked cross-attention for temporal alignment + adaptive gating for training stability
- Decoupled-Then-Joint Training: train separate R2V and A2V models first, then merge via weight interpolation (0.6, 0.4) for joint training
- HOIVG-Bench: 135 curated samples with comprehensive metrics across text alignment, reference consistency, pose accuracy, audio-visual sync, and video quality

## Method

**Unified Channel-wise Conditioning**: Pose rendered as RGB video, encoded via VAE. Reference images also encoded. Noisy video tokens augmented with N' pseudo-frame tokens. All conditions injected via channel-wise concatenation: xin = Concat([x'|x_t], [r|p], [m'|m]). Reference reconstruction loss on pseudo-frames ensures semantic detail preservation.

**Gated Local-Context Attention**: Audio processed by Wav2Vec 2.0 with layer merging. Sliding window (size=5) with stride=4 for temporal context aggregation. Masked attention restricts video tokens to attend only to corresponding audio segments. Learnable gating vector g initialized to 1e-5 modulates audio injection, inserted only in dual-stream blocks (+2.5% model scale).

**Decoupled-Then-Joint Training**: Phase 1 trains specialized R2V model (discards audio modules) and A2V model (incorporates first-frame image). Phase 2 merges models via weight interpolation (0.6 A2V, 0.4 R2V), trains on full RA2V dataset, then fine-tunes on high-quality RAP2V subset with pose introduced only in final stage.

## Results

On HOIVG-Bench (135 samples, 5-second 720p portrait videos):

**R2V Setting**:
- FaceSim: 0.874 (vs Phantom-14B's 0.876), NexusScore: 0.389
- Video quality: AES 0.468, VQ 11.12, MQ 5.885 (best overall)

**RA2V Setting**:
- Sync-C: 8.612, Sync-D: 7.608 (best audio-visual sync)
- FaceSim: 0.810, NexusScore: 0.369

**RP2V Setting**:
- AKD: 0.174 (best pose accuracy), PCK: 0.460

Only model supporting full RAP2V (all 4 conditions). Human evaluation shows strong preference (majority wins in both RA2V and RP2V settings).

## Limitations & Future Work

Current limitation: generation capped at 10 seconds. Future directions include:
- Extending to longer video generation
- Adding more modalities (e.g., depth, haptic)
- Improving efficiency for real-time applications
