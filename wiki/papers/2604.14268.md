# HY-World 2.0: A Multi-Modal World Model for Reconstructing, Generating, and Simulating 3D Worlds

**Published**: 2026-04-14 | **Added**: 2026-04-17 | **arXiv**: [2604.14268](http://arxiv.org/pdf/2604.14268)

## Summary

HY-World 2.0 addresses the long-standing gap between 3D world generation (synthesizing explorable scenes from sparse inputs) and world reconstruction (recovering geometric structures from dense multi-view inputs). The framework introduces a four-stage pipeline that unifies reconstruction and generation, producing navigable 3D assets in approximately 10 minutes.

The paper presents four key innovations: HY-Pano 2.0 with adaptive perspective-to-equirectangular transformation, WorldNav for semantic-aware trajectory planning, WorldStereo 2.0 operating in keyframe latent space with consistent memory mechanisms, and WorldMirror 2.0 with normalized position encoding for flexible resolution inference.

## Key Contributions

- **Unified reconstruction-generation framework**: Bridges the gap between sparse-input generation and dense-input reconstruction
- **Four-stage pipeline**: Panorama → Trajectory → Expansion → Composition
- **Real-time completion**: ~10 minutes for end-to-end world generation
- **Open-source SOTA**: Results comparable to closed-source Marble model

## Method

**Four-Stage Pipeline:**

1. **Panorama Generation (HY-Pano 2.0)**: Adaptive perspective-to-equirectangular transformation for scene initialization
2. **Trajectory Planning (WorldNav)**: Five heuristic modes (surrounding, reconstruct-aware, depth-aware, structure-aware, hybrid) for optimal scene coverage
3. **World Expansion (WorldStereo 2.0)**: Keyframe latent space with Global-Geometric Memory and Spatial-Stereo Memory++ for consistent stereo expansion
4. **World Composition (WorldMirror 2.0 + 3DGS)**: Normalized position encoding enables flexible resolution inference; MaskGaussian handles scene composition

**Key Innovations:**
- WorldNav: Semantic-aware trajectory planning with five heuristic modes
- WorldStereo 2.0: Memory mechanisms ensure cross-keyframe consistency
- WorldMirror 2.0: Normalized position encoding removes resolution-dependent artifacts

## Limitations & Future Work

- Quality dependent on input sparsity/density
- Trajectory planning heuristics may not cover all scene types
- Real-time constraint (~10 min) may limit complexity
