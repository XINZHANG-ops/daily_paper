## Summary

SIMPLESTREAM challenges a prevailing assumption in streaming video understanding: that increasingly complex memory mechanisms (memory banks, retrieval, compression modules) are necessary for strong performance. The paper demonstrates that a deliberately minimal baseline—feeding only the most recent N frames directly to an off-the-shelf vision-language model—already matches or surpasses published streaming methods that rely on elaborate memory architectures.

The key insight is a **perception-memory trade-off**: adding more historical context can improve memory-oriented behavior (episodic recall), but typically at the cost of real-time present-scene perception. SIMPLESTREAM exploits this by preserving an undiluted view of the latest frames, which aligns with the strongest capability of modern VLMs. With only 4 recent frames, SIMPLESTREAM achieves 67.7% average accuracy on OVO-Bench and 80.59% on StreamingBench, outperforming HERMES (the strongest prior streaming method at 59.2% on OVO-Bench) while maintaining the lowest peak GPU memory and competitive latency among all compared streaming baselines.

Controlled ablations reveal that longer context is not uniformly beneficial—it does not increase monotonically with model scale, and the preferred window size varies across backbone families and model sizes. The paper argues that future streaming benchmarks should disaggregate perception-versus-memory metrics, so gains from added complexity can be evaluated more clearly rather than assumed.

## Key Contributions

- **Strong Minimal Baseline**: SIMPLESTREAM feeds only the last N frames to an off-the-shelf VLM without architectural modifications, memory modules, or additional training—achieving 67.7% on OVO-Bench, surpassing HERMES by 8.5pp
- **Perception-Memory Trade-off Analysis**: Quantifies that external memory mechanisms improve memory-oriented tasks (EPM, ASI) but consistently degrade real-time perception; proposes ΔP (real-time perception change) as a key metric
- **Model-Scale Dependent Context Utility**: Demonstrates that optimal recent-window size varies non-monotonically across backbones (Qwen2.5-VL-72B prefers 16 frames; Qwen3-VL-30B-A3B peaks at 4 frames), challenging assumptions that larger models uniformly benefit from longer context
- **Benchmark Design Critique**: Identifies that OVO-Bench's macro-average mixes perception and memory tracks unequally (6 Real-Time vs 3 Backward tracks), and that HLD (Hallucination Detection) is misaligned as a memory metric
- **Efficiency Leadership**: Maintains lowest peak GPU memory across all frame budgets (35-38GB at 16-256 observed frames vs 490GB for Dispider at 256 frames) and competitive latency (second-lowest TTFT after HERMES)

## Method

**SIMPLESTREAM Formulation**: Given a video stream as frames f_1...f_t and a query q_t at time t, SIMPLESTREAM(t) = VLM({f_{t-N+1}, ..., f_t}, q_t). Frames outside the sliding window are discarded; per-query memory and computation remain bounded by N and do not grow with stream length.

**Design Principles**:
- No architectural modifications to the VLM backbone
- No memory modules, retrieval, or compression beyond the recent window
- Inference-time input policy applied to off-the-shelf models
- Samples visible stream at 1 fps; feeds only last N ∈ {2, 4, 8} frames

**Backbones**: Qwen2.5-VL-7B-Instruct and Qwen3-VL-8B-Instruct evaluated across window sizes N ∈ {2, 4, 8, 16}.

**Evaluation Protocol**: All models evaluated under unified OVO-Bench protocol where each question is answered using only the video prefix available up to query time. Compares against 6 offline video LLMs (Qwen2.5-VL-7B, LLaVA-OneVision-7B, InternVL2-8B, etc.) and 7 streaming baselines (HERMES, StreamForest, TimeChat-Online, Flash-VStream, etc.).

## Results

**Main Results on OVO-Bench** (Real-Time Visual Perception / Backward Tracing Average):
| Configuration | OVO-Bench Avg | StreamingBench |
|--------------|--------------|----------------|
| SIMPLESTREAM (Qwen3-VL-8B, 4f) | **67.7%** | **80.59%** |
| HERMES-7B (4K tokens) | 59.2% | 79.44% |
| StreamForest-7B | 56.6% | 77.26% |
| Offline Qwen2.5-VL-7B | 52.3% | 73.31% |

**Perception vs Memory** (relative to SIMPLESTREAM Qwen2.5-VL-2f anchor):
- HERMES: Memory Gain +2.4, Perception Cost -6.0
- StreamForest: Memory Gain +8.9, Perception Cost -13.8
- Visual-RAG: Memory Gain +6.6, Perception Cost -4.9

**Model Scale Effects**: Optimal window size varies by model family—Qwen2.5-VL-72B peaks at 16f while Qwen2.5-VL-32B peaks at 4f; Qwen3-VL-32B prefers 8f.

**Efficiency**: SIMPLESTREAM-4f maintains flattest memory curve (35-38GB across 16-256 observed frames) and achieves second-lowest TTFT after HERMES.

## Limitations & Future Work

- **Backbone Dependence**: Conclusions coupled to Qwen2.5-VL/Qwen3-VL family capabilities; transfer to broader model families with different pretraining or visual encoders unverified
- **Positioning as Strong Baseline Study**: Paper is deliberately a strong baseline study, not a new streaming architecture proposal; does not introduce memory mechanisms or retrieval designs
- **Benchmark Design Issues**: Current benchmarks mix perception and memory tracks unequally; aggregate scores may not faithfully reward long-term memory as intended
- **Future Research Directions**: Recent-first/history-on-demand principle (preserve recent context by default, access history on-demand); memory modules should be evaluated by both recall gains and perception costs
