# OneVL: One-Step Latent Reasoning and Planning with Vision-Language Explanation

**Published**: 2026-04-19 | **Added**: 2026-04-21 | **arXiv**: [2604.18486](http://arxiv.org/pdf/2604.18486)

## Summary

OneVL addresses a fundamental tension in Vision-Language-Action (VLA) autonomous driving: Chain-of-Thought (CoT) reasoning improves trajectory prediction quality but imposes prohibitive inference latency for real-time deployment. Existing latent CoT methods compress reasoning into continuous hidden states but consistently underperform explicit CoT because purely linguistic representations compress symbolic abstractions rather than causal scene dynamics. OneVL introduces dual-modal auxiliary decoders that supervise compact latent tokens with both language CoT reconstruction and future-frame visual prediction (world model supervision), ensuring latents encode genuine causal structure. At inference, decoders are discarded and latent tokens are prefilled in a single parallel pass, matching answer-only prediction speed while retaining interpretability in both vision and language.

## Key Contributions

- First latent CoT method to surpass explicit CoT while matching answer-only prediction latency (4.46s vs 4.49s on NAVSIM)
- Dual-modal supervision framework combining language reasoning and visual world model prediction
- Three-stage training pipeline for stable joint optimization of main model and auxiliary decoders
- Optional MLP head deployment achieving 0.24s inference (5.4% of autoregressive model latency)

## Method

OneVL builds on Qwen3-VL-4B-Instruct as backbone and introduces two types of latent tokens: visual latent tokens (Zv, 4 tokens) encoding spatial-temporal reasoning about future scene state, and language latent tokens (Zl, 2 tokens) encoding linguistic CoT reasoning. During training, hidden states at latent positions are routed to dual auxiliary decoders: a language decoder reconstructs CoT text via cross-entropy loss, while a visual decoder predicts future-frame visual tokens (at +0.5s and +1.0s) using an IBQ visual tokenizer with 131K codebook. The three-stage pipeline progressively aligns latents: Stage 0 warms up the main model with latent tokens, Stage 1 freezes the main model and trains auxiliary decoders for stable alignment, and Stage 2 jointly fine-tunes all components. At inference, both decoders are discarded and latent tokens are prefilled into the prompt context for single-pass generation.

## Results

| Benchmark | Metric | OneVL Result |
|-----------|--------|--------------|
| NAVSIM | PDM-score | 88.84 |
| ROADWork | ADE (px) | 12.49 |
| Impromptu | ADE (m) | 1.34 |
| Alpamayo-R1 | ADE (m) | 2.62 |

OneVL achieves state-of-the-art accuracy across all four benchmarks while delivering 1.5-2.3× faster inference than explicit autoregressive CoT. The visual auxiliary decoder's future-frame prediction serves as a rigorous test of generalization—predicting unseen scene configurations requires a robust causal model rather than pattern memorization.

## Limitations & Future Work

- The approach requires three-stage training which increases complexity
- Visual decoder prediction is training-only (discarded at inference) so intermediate visual explanations are not available in deployment
- Future work could explore keeping visual decoder at inference for real-time scene monitoring
