## Summary

SkillClaw introduces a framework for collective skill evolution in multi-user agent ecosystems, addressing the fundamental limitation that current LLM agent skills remain static after deployment. In practical terms, this means when users encounter similar workflows, tool usage patterns, and failure modes across different sessions, the system does not improve—it forces each user to independently rediscover solutions that could be shared system-wide. The core innovation is treating cross-user and over-time interactions as the primary signal for improving skills through a centralized evolution architecture that aggregates interaction trajectories, analyzes them with an autonomous agentic evolver, and synchronizes improved skills back to all agents.

The framework operates through a closed-loop pipeline: agents deployed across different users independently generate interaction sessions during normal usage; these sessions preserve full action-feedback causal chains including user prompts, agent actions, tool calls, intermediate feedback, and final responses; trajectories are aggregated across users and grouped by referenced skills, forming a shared evidence base that exposes consistent success patterns and recurring failure modes; an agentic evolver analyzes each skill-specific group and performs evidence-driven updates via refinement or creation while preserving validated behaviors from successful executions; the updated skill repository is synchronized back to all agents. The evolver uses structured prompts that instruct it to analyze session evidence and decide among four actions: improve_skill, optimize_description, create_skill, or skip.

Key to the system is the validation mechanism that ensures monotonic improvement: during nighttime, candidate skill updates are deployed to available user environments and evaluated under real execution conditions using the same full toolchain. Only updates demonstrating better performance than currently deployed versions are accepted; rejected updates are retained as candidates but not deployed. This guarantees the deployed skill pool never degrades while enabling the system to accumulate knowledge from distributed user interactions.

## Key Contributions

- **Collective Skill Evolution**: Enables knowledge from individual user interactions to contribute to a shared and continuously improving skill ecosystem, with improvements discovered in one context propagating to all users without additional effort from anyone.

- **Agentic Evolver Paradigm**: Skill updates are produced through open-ended reasoning by an LLM agent equipped with a structured harness, rather than predefined update rules, enabling flexible and context-aware improvements that can handle previously unseen failure modes.

- **Structured Session Aggregation**: Converts isolated interaction sessions into structured evidence by preserving full causal chains (prompt→action→feedback→response), grouping by referenced skills to enable cross-user reasoning about behavioral boundaries.

- **Conservative Validation with Monotonic Guarantees**: Nighttime validation ensures only performance-improving updates are deployed, guaranteeing the skill pool never degrades while forming a closed-loop: Interaction→Evidence→Evolution→Validation→Deployment.

- **Comprehensive Benchmark Results**: Demonstrates substantial improvements on WildClawBench: +52% in Search & Retrieval, +88% in Creative Synthesis, +33% in Safety & Alignment, and +12% in Social Interaction categories.

## Method

The system architecture comprises four interconnected components. First, the multi-user interaction layer connects independently deployed agents through a common skill repository, where each agent produces interaction sessions during normal usage that are recorded and uploaded as shared evidence. Second, the evidence aggregation layer structures individual sessions to preserve causal information, then groups them by referenced skills: G(s) = {τ_i | s ∈ K_i} for skill s, with sessions using no skill placed in G(∅). This grouping exposes behavioral boundaries—where a skill works versus breaks—enabling reliable skill evolution. Third, the agentic evolver receives grouped session evidence, current skill definitions, and permitted actions (refine, create, skip), using open-ended LLM reasoning to analyze success and failure patterns jointly, with successful sessions defining invariants that must be preserved and failed sessions defining targets for correction. Fourth, the validation layer deploys candidate updates to user environments during nighttime, comparing against current deployed versions under the same tasks and environment conditions, accepting only improvements.

The skill update process follows Algorithm 1: convert sessions to structured evidence, group by referenced skills, then for each group use the agentic evolver to analyze recurring patterns, select an evolution action, generate candidate updates if evidence supports modification, apply conservative editing and validation, and merge approved updates. For G(∅) sessions, the evolver identifies missing but reusable procedures that could become new skills.

The evolution harness provides structured inputs without constraining reasoning: it supplies the skill manifest, session data grouped by skill, version history, and evaluation criteria. The evolver reasons over evidence and decides actions using detailed editing principles that prioritize preserving correct API information, avoiding agent-level advice in skills, and preferring skip over speculative edits when evidence is weak.

**Skill Evolution Patterns by Category**:
- **Social Interaction**: Workflow executability improvement; effective skill was 03_task6 (Cross-dept Slack summarization) which was refined and accepted on Night 1
- **Search & Retrieval**: Staged improvement through input reliability first (validate-file-existence accepted Night 1), then constraint-aware planning later
- **Creative Synthesis**: Environment setup skills most impactful (validate-tmp-workspace-inputs accepted Night 1), content generation pipelines evolved but not surpassing early best
- **Safety & Alignment**: Reliability-focused updates with git-push-with-auth-fallback accepted and refined across Nights 1-4

**Case Studies**:
- **Slack message analysis**: Original skill had wrong API port and relied on trial-and-error; evolved skill fixed port, added selective retrieval using previews, and specified correct output paths
- **ICCV oral paper analysis**: Original used heuristic matching; evolved skill added explicit first-affiliation check and targeted re-verification for noisy extraction
- **SAM3 inference under incomplete environments**: Original assumed all files present; evolved skill added environment precheck and nearby asset discovery

## Limitations & Future Work

The paper acknowledges that this study represents a small-scale test of collective skill evolution with limited user queries, feedback signals, and interaction depth. Despite these constraints, consistent performance gains demonstrate effectiveness in realistic settings. Scaling up the number of users, extending the time horizon, and introducing more diverse tasks and validation conditions are likely to further enrich evolution trajectories and improve system performance. The agentic evolver's reasoning quality depends on the underlying LLM's capabilities, and the conservative editing principles may slow evolution for edge cases that require more aggressive changes. Future work includes investigating more efficient validation strategies and exploring agentic evolver architectures specifically optimized for skill evolution tasks.
