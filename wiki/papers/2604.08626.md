# WildDet3D: Scaling Promptable 3D Detection in the Wild

**Published**: 2026-04-08 | **Added**: 2026-04-13 | **arXiv**: [2604.08626](http://arxiv.org/pdf/2604.08626)

## Summary

WildDet3D addresses the challenge of open-vocabulary monocular 3D object detection for general-purpose spatial understanding in real-world environments. The paper makes two key contributions: (1) a unified geometry-aware architecture (WildDet3D) that accepts text, point, and box prompts while optionally incorporating depth signals at inference time, and (2) WildDet3D-Data, the largest open 3D detection dataset to date with over 1M images across 13.5K categories—138x more than existing benchmarks. The architecture employs dual-vision encoders (image + RGBD) with a depth fusion module, a promptable detector supporting four complementary prompt types, and a deeply-supervised 3D detection head with unambiguous rotation normalization.

The system achieves state-of-the-art results across multiple settings: 34.2 AP3D on Omni3D with text prompts (36.4 with oracle box prompts), 22.6 AP3D on the new WildDet3D-Bench, and 40.3/48.9 ODS zero-shot on Argoverse 2/ScanNet. Notably, incorporating depth cues at inference time yields +20.7 AP average gain across settings, while training requires only 6-10x fewer epochs than prior methods (12 epochs vs 80-120 epochs).

## Key Contributions

- **Unified Promptable Architecture**: Single architecture accepting text, point, box, and exemplar prompts with optional depth input at inference time
- **Dual-Vision Encoders**: Image encoder (ViT-H pretrained on segmentation) + RGBD encoder (DINOv2 ViT-L/14 pretrained on depth completion) with ControlNet-style depth fusion
- **WildDet3D-Data**: 1M images, 13.5K categories (138x more than Omni3D), human-verified via multi-model candidate generation + VLM + human selection
- **Efficient Training**: 12 epochs vs 80-120 for competing approaches, with auxiliary 2D detection and depth estimation heads for multi-source supervision
- **Optional Depth Handling**: Graceful degradation to monocular mode when depth unavailable; substantial gains (+20.7 AP) when depth is present

## Method

**Architecture Overview:**
The model takes an RGB image, optional camera intrinsics, optional depth map, and a user-specified prompt to predict 3D bounding boxes. Each box consists of 3D center (meters), physical dimensions (w, h, l in meters), orientation (SO(3) with unambiguous rotation normalization), and confidence score.

**Dual-Vision Encoder:**
Two encoders operate in parallel: (1) Image encoder: ViT-H with SimpleFPN neck from SAM, providing high-resolution semantic features (last 4 of 32 ViT blocks fine-tuned); (2) RGBD encoder: DINOv2 ViT-L/14 accepting 4-channel RGBD input at 686x686 resolution, producing depth latents through ConvStack neck (last 3 of 24 blocks fine-tuned). Training uses stochastic depth strategy: 70% monocular (zero depth), 20% patch-masked depth, 10% full depth.

**Depth Fusion Module:**
ControlNet-style residual design merges depth latents into image features via element-wise addition. The convolution is zero-initialized at training start (identity mapping), preserving pretrained feature distributions while gradually learning depth contributions.

**Promptable Detector:**
Four prompt types supported:
- **Text prompt**: Category name tokenized via CLIP BPE, encoded by 24-layer causal text Transformer
- **Point prompt**: 2D coordinates with positive/negative labels, geometry encoder with sinusoidal positional encoding
- **Box prompt**: 2D bounding box with ROI-aligned features
- **Exemplar prompt**: Visual exemplar for category detection via multi-target matching

Per-prompt batching aggregates all images containing each unique text category for balanced learning.

**3D Detection Head:**
Deep supervision with equal weights across L transformer decoder layers. Each layer aggregates camera geometry via spherical harmonics ray features (8th-order RSH encoding) and depth latents through cross-attention. Unambiguous rotation normalization avoids elevation/azimuth ambiguity in SO(3) orientation.

**Auxiliary Heads:**
Auxiliary 2D detection and depth estimation heads provide complementary supervision that substantially improves 3D performance.

## Results

**Omni3D Benchmark:**
| Setting | AP3D |
|---------|------|
| Text prompts | 34.2 |
| Oracle box prompts | 36.4 |
| Prior best (3D-MOOD) | ~2.3 |

**WildDet3D-Bench (700+ open-vocabulary categories):**
| Setting | AP3D |
|---------|------|
| Text prompts | 22.6 |
| Box prompts | 24.8 |
| With ground-truth depth | 41.6 (text), 47.2 (box) |

**Zero-Shot Transfer:**
- Argoverse 2: 40.3 ODS
- ScanNet: 48.9 ODS

**Depth Contribution:**
Average +20.7 AP gain when depth is available at inference time across all settings.

**Training Efficiency:**
- WildDet3D: 12 epochs
- Prior methods: 80-120 epochs
- 6-10x reduction in training compute

## Limitations & Future Work

The paper focuses on single-image monocular 3D detection; video-based temporal integration could further improve stability and tracking. The dataset construction relies on existing 2D annotations and VLM-human verification, which may introduce biases. Future work could explore end-to-end 3D annotation pipelines and extend to more diverse sensor modalities (e.g., event cameras, radar).
