# CodeTracer: Towards Traceable Agent States

**Published**: 2026-04-12 | **Added**: 2026-04-14 | **arXiv**: [2604.11641](http://arxiv.org/pdf/2604.11641)

## Summary

CodeTracer is a tracing architecture for debugging and failure localization in code agents that autonomously interact with software repositories. The system converts heterogeneous agent run directories into structured hierarchical traces and performs failure onset localization to pinpoint the failure origin and its downstream chain. The key insight is that early missteps can trap agents in unproductive loops or cascade into hidden error chains, making it hard to tell when and why an agent goes off track. CodeTracer achieves up to 48% macro F1 in step-level failure localization while reducing token costs, and its diagnostic signals can be fed back into agents via reflective replay to recover originally failed runs.

## Key Contributions

- Three-stage pipeline: evolving extraction (adaptive parsers), tree indexing (hierarchical trace trees), and diagnosis (failure onset localization)
- CodeTraceBench: 3,326 annotated trajectories spanning 5 backbones and 4 agent frameworks
- Discovery of "evidence-to-action gap": agents retrieve relevant info but fail to translate it into correct state-changing actions
- Reflective replay: using diagnostic signals to recover failed runs under matched budgets

## Method

The pipeline consists of three stages:

1. **Evolving Extraction**: Parses heterogeneous run artifacts through adaptive parsers. Checks a registry of existing parsers; if no match, synthesizes and registers a new one. Emits normalized step records with typed fields (action, observation, diff, verification outcome).

2. **Tree Indexing**: Reconstructs trajectories as hierarchical trace trees. Exploration steps (inspect without modifying) stay under current state node; state-changing steps induce transitions to child states. Each node annotated with intent and outcome summary.

3. **Diagnosis**: Traverses the tree, issues structured evidence queries, outputs failure-responsible stage, error-relevant steps, and compact evidence set.

Data was collected from SWE-bench Verified, SWE-bench Pro, MultiSWE-bench, SWE-PolyBench, and TerminalBench using four agent frameworks (SWE-Agent, MiniSWE-Agent, OpenHands, Terminus 2) with five backbones (Claude-sonnet-4, GPT-5, DeepSeek-V3.2, Qwen3-Coder-480B, Kimi-K2-Instruct).

## Results

Main results on CodeTraceBench (macro F1, step level):
- CodeTracer with Claude-sonnet-4: 46.57% F1, 56.8k tokens
- CodeTracer with GPT-5: 48.02% F1, 31.1k tokens
- CodeTracer with DeepSeek-V3.2: 46.14% F1, 44.6k tokens

Key behavioral findings:
- **Evidence-to-action gap**: ineffective steps nearly double from solved (22%) to unsolved (40%)
- **Overengineering**: SWE-Agent and OpenHands consume 2× tokens of MiniSWE-Agent for only +2.4-5.5pp success gain
- **Stage-dependent errors**: environment errors concentrate early; mislocalized edits dominate later stages
- **Success saturates**: additional iterations amplify unproductive loops rather than enabling recovery

Reflective replay consistently improves Pass@1 across all backbones on originally failed runs.

## Limitations & Future Work

The system requires agent run directories with specific artifacts — it cannot diagnose from sandboxed execution alone. Future work could extend to:
- Real-time failure prediction before errors cascade
- Integration with industrial agents beyond academic frameworks
- Automatic repair suggestions based on diagnostic signals
