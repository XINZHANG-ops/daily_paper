# CodeTracer: Towards Traceable Agent States

**Published**: 2026-04-12 | **Added**: 2026-04-14 | **arXiv**: [2604.11641](http://arxiv.org/pdf/2604.11641)

## Summary

CodeTracer addresses debugging and failure localization in LLM-powered code agents. The core problem is that early missteps can trap agents in unproductive loops or cascade into hidden error chains, making it difficult to identify where and why an agent goes off track.

The solution is a three-stage pipeline: (1) **evolving extraction** parses heterogeneous run artifacts through adaptive parsers, (2) **tree indexing** reconstructs trajectories as hierarchical trace trees with persistent memory, (3) **diagnosis** localizes failure origins through evidence retrieval and root-cause tracing.

## Key Contributions

- **Three-stage pipeline**: Evolving extraction → tree indexing → diagnosis
- **Evidence-to-action gap identification**: Despite gathering relevant information, agents fail to translate evidence into correct actions
- **CodeTraceBench**: 3,326 annotated trajectories across five backbones and four agent frameworks
- **Up to 48% macro F1**: In step-level failure localization while reducing token costs

## Method

Three-stage pipeline: (1) Evolving extraction parses heterogeneous run artifacts through adaptive parsers, (2) Tree indexing reconstructs trajectories as hierarchical trace trees with persistent memory, (3) Diagnosis localizes failure origins through evidence retrieval and root-cause tracing. Reflective replay uses diagnostic signals to recover originally failed runs.

## Results

- **Step-level failure localization**: Up to 48% macro F1
- **Token cost reduction**: Diagnostic signals reduce token consumption
- **Reflective replay**: Successfully recovers originally failed runs under matched budgets
- **CodeTraceBench**: 3,326 annotated trajectories spanning five backbones and four agent frameworks
